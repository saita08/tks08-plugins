# parallel-universe

> 「独立した複数案を出して選ぶほうが、一案を磨くより速い」
> 「知ってる。全人類が知ってる」
> 「じゃあやろう」
> 「面倒だからやらない」

こうして人類は、正しいと知っている手法を毎回見送り、一本の草稿にしがみついて三時間かけて磨き、磨き終わってから「最初の方向が間違ってた」と気づきます。`parallel-universe` は、この悲劇を一回のコマンドにまとめて肩代わりします。

## これは何？

一つの課題を、複数の並行世界に分岐させるプラグインです。それぞれの宇宙では、同じ作業者の別バージョンが、互いの存在を知らないまま、別々の角度から同じ課題に挑みます。片方が最小実装を突き詰めている隣で、もう片方は堅牢性に命を懸けている。三人目は性能、四人目は使い心地、五人目は「そもそもこの課題設定が間違っているのでは」と疑いにかかる。

全員が挑み終わったら、三人の審査員が登場します。それぞれ別のレンズ、正しさ・簡潔さ・意図適合、を通して全案を採点する。一人の審査員に三つの基準を渡すと、こっそり一つだけを贔屓して残りを黙殺するので、そうさせません。三人が別々の勝者を推したら、その食い違いこそがあなたの見るべきトレードオフです。

最後に優勝案を土台として、次点たちの良い一手を接ぎ木した合成案を提示します。混ぜてぼかすのではなく、接ぐ。どこから来た一手かを名指しで示すので、あなたは一手ずつ承認できます。

## なぜ一案磨きに勝つのか

一本の草稿は、書いた瞬間から錨になります。以降のすべての修正は、目標ではなくその錨と交渉することになる。「ここをこう直して」の全部が、最初の形からの相対距離で語られる。だから草稿は、それが良かろうと悪かろうと、思考をその形に縛りつけます。

独立した五本の草稿には、共有された錨がありません。だから選択は、草稿同士の相対ではなく、目標そのものに対して下される。これがN倍のトークンを払う理由であり、同時に各案が独立でなければならない理由でもあります。互いが見えた瞬間、五本は一つの形に collapse し、あなたは一枚の草稿に五回払ったことになる。

## 仕組み

このプラグインは Claude Code の Workflow を動かします。会話でエージェントを率いるのではなく、プログラムで統率する仕組みです。スラッシュコマンドから Workflow を呼ぶのは公式に認められた使い方なので、堂々と使います。

コマンドが組み立てるスクリプトは三つのフェーズを持ちます。Generate フェーズは、N体の作業者をそれぞれ隔離された worktree で並列に走らせる。worktree の隔離は、並列にファイルを書く作業者たちが衝突しないためであり、それ以上に、誰も隣の書きかけを盤面から盗み見できないための封印です。Judge フェーズは三人の審査員が構造化スキーマで採点する。Synthesize フェーズが接ぎ木案を出す。

全部が隔離された worktree の中で起きるので、あなたの本物の作業ツリーは最後まで無傷です。適用はあなたが勝者を選んでから。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/parallel-universe` | 課題を複数の並行世界に分岐させ、審査し、接ぎ木合成案まで提示する |
| Skill | `tournament-doctrine` | いつ発散が磨き込みに勝つか、独立性をどう守るか、審査レンズの設計、そしてWorkflowの組み方の教義 |

## 使い方

```
/parallel-universe <課題> [--n 数]
```

`--n` は宇宙の数で、既定は3、最大は5です。課題が一行の意図しかない場合、五倍のトークンを曖昧な目標に注ぐ前に、最も攻め方を左右する質問を一つ二つ返します。曖昧な目標に発散を仕掛けても、買えるのは同じ誤解のN通りのバリエーションだけだからです。

走り終わると、勝者の diff 要約、各案を各レンズで採点した審査表、そして接ぎ木案が報告されます。あなたが選ぶまで、本物のツリーには何も着地しません。優勝宇宙をそのまま採るか、合成案を採るか、全部捨てるか。全部捨てても、それは無駄ではありません。N個の方向を却下したという事実が、あなたの本当に欲しいものの輪郭を削り出しています。

## いつ使い、いつ使わないか

発散が磨き込みに勝つのは、目標が「見れば分かるが、先には言葉にできない」ときです。きれいなAPIの形、本当に助けになるエラーメッセージ、モジュールをやっと腑に落とす refactor。仕様が言葉ではなくあなたの判断の中にあり、具体的な候補への反応としてしか出てこない種類の目標。ここでこそ、五本の探り針があなたの判断を高速に写し取ります。

逆に、正解を今この場で書き下せる課題には向きません。日付のパース、null チェックの追加、シンボルの一括改名。正解が一つに決まっている課題に五宇宙を仕掛けると、同じコードの五つのコピーと、無意味な順位表が手に入ります。この見極めはコマンド自身が行い、割に合わないと判断したら、五回課金する代わりに正直にそう言って止まります。

## よくある質問

**Q: 結局いくらかかるの？**
A: N体ぶんのトークンに、三人の審査員と一回の合成が乗ります。五宇宙なら、ざっと六から八体ぶん。安くはありません。このプラグインは、その多重課金と引き換えに、難しい目標への収束速度を買う道具です。安さを売りにしたことは一度もありません。

**Q: 五体で挑む価値のない課題に使ったら？**
A: 五通りの同じ答えが手に入ります。全員が同じ正解に収束し、審査員は五つの同一案に順位をつける。あなたは一つの答えに五回払ったことになります。だからコマンドは、その課題が本当に発散を欲しているかを先に判定します。欲しがっていなければ、走らせません。

**Q: 却下した案が、合成のときにこっそり戻ってこない？**
A: 戻しません。一度却下された方向は、目標についてのデータです。それが新しい服を着て再登場することは、審査が明示的に禁じています。あなたが捨てたものは、捨てたまま記録されます。

**Q: 隣の宇宙は本当に互いを知らないの？**
A: 知りません。各作業者のプロンプトは、他の宇宙の存在を一言も語らず、自分が唯一の挑戦者だと信じています。競争していると知った作業者は、守りに入って無難な中庸に寄る。一人だと思っている作業者だけが、自分の角度に全部を賭けます。

観測されるまで、どの宇宙も確定しません。あなたが選んだ瞬間、残りは静かに折り畳まれます。
