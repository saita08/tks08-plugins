# tengan

> 「実装しました。動くはずです」
> 「見た目は確認した?」
> 「……いえ、目で見てはいません」

世界中のClaudeが、この会話を毎日交わしています。コードは書けた、ロジックは通っている、だから動くはずだ。その「はず」の先で、ボタンのラベルは見切れ、空状態は真っ白な虚無を晒し、モバイルではレイアウトが崩壊している。誰も見ていないからです。`tengan` は、その最後の一歩に目を開かせるプラグインです。

## これは何？

天眼通は、遠く千里の彼方までを見通す仏教の神通力です。このプラグインが授けるのは、その現代版です。成果物のスクリーンショットを撮り、Claude自身がその画像を実際に見て、意図との差を判定する。人間の目を待たずに、視覚検証のループを閉じます。

奇妙な話ですが、Claudeは画像をネイティブに読めます。読めるのに、UIの仕事はなぜか「実装した、あとは人間が見てくれ」で終わっていました。目があるのに閉じたまま歩いていたのです。tenganは、その目を開けさせるだけです。それだけで、見えていなかったものが見えます。

## 何を見通すのか

視覚面はシミュレータの中だけにあるのではありません。tenganの視界は、ピクセルを生むものすべてに及びます。

- **iOSシミュレータ**: `xcrun simctl` で起動中の画面を撮影します。iPhone SEでは収まっていたレイアウトがPro Maxで間延びしていることに、実機を触る前に気づきます
- **Webアプリ**: ヘッドレスブラウザでモバイルとデスクトップの両方を撮ります。レスポンシブ崩れは、まさに幅の違うところに隠れて人間を待ち伏せしています
- **Remotion動画**: `npx remotion still` で任意フレームの静止画を抜きます。1フレームでは動画の何もわからないので、冒頭・中間・終端を撮ります
- **一般の動画ファイル**: ffmpeg で複数地点のフレームを抽出します。テキストが画面に入りきる前に消えていないか、退場したはずの要素が居残っていないかは、フレームを並べて初めて見えます
- **Chrome拡張**: Playwright に拡張を読み込ませて、ポップアップもオプション画面も撮ります
- **デプロイ済みWebアプリ**: 公開URLをヘッドレスで撮影します。Google Apps Script の `/exec` も、ただのWebページとして同じように見通せます
- **画像・PDF・SVG**: 撮影すら要りません。Read で直接見ます。最小の面には、儀式もいりません

## どう見通すのか

tenganは二段構えです。まず目を開き、それから見ます。

### 開眼 — プロジェクトを調べ、撮り方を書き留める

`/tengan:open` は、プロジェクトを調査して視覚面を検出します。何があればiOSプロジェクトなのか、何があればRemotionなのか、証拠から判定します。名前で決めつけません。面ごとの撮影レシピ、つまり撮影コマンド・前提・撮るべき複数状態を `.claude/tengan.local.md` に書き込みます。後のセッションが、この調査を覚えていなくても、このファイルを読むだけで全面を撮れるように書きます。

撮影に必要な道具が無いときは、勝手に入れません。代替の撮り方と、ユーザーが選んで実行できるインストール提案を書き添えて、判断はユーザーに残します。神通力とはいえ、他人のマシンに黙って何かをインストールする力ではないのです。

### 撮って、見て、判定する

`/tengan:look` は、レシピに従って撮影し、**Readで画像を実際に見て**、判定を報告します。ここが肝です。「スクリーンショットを撮った」で止まりません。ピクセルを開いて、意図との差を見ます。撮っただけで誰も見ないスクリーンショットこそ、このプラグインが終わらせにきた失敗そのものです。

崩れが見つかり、直し方が明白なら、直して、その面をもう一度撮って、また見ます。合致するまで回します。判定が趣味やデザインの選択に及ぶところで、ループは止まります。スクリーンショットと具体的な指摘を添えて、判断をユーザーに返します。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/tengan:open` | プロジェクトを調査して視覚面を検出し、面ごとの撮影レシピを `.claude/tengan.local.md` に書く |
| Command | `/tengan:look` | レシピに従って撮影し、画像を実際に見て、意図との差を判定し、修正ループを回す |
| Skill | `close-the-visual-loop` | UIに関わる仕事を「動くはず」で終えず、撮って見て判定する、という判断の教義 |

## 使い方

まず一度だけ、目を開きます。

```
/tengan:open
```

プロジェクトの視覚面が検出され、レシピが書き出されます。あとは見たいときに:

```
/tengan:look
```

面や状態を指定すれば、そこだけを見ます。省略すれば、レシピの全面を、記録された全状態で撮って見ます。ハッピーな1状態だけ撮って「検証済み」と言い張る癖こそ、このプラグインが置き換えるものです。

スキルの方は、あなたが何も呼ばなくても働きます。UIや見た目に関わる仕事を終えようとして、しかしピクセルを一枚も見ていないとき、「それ、見てから終わりませんか」と判断が差し込まれます。

## よくある質問

**Q: コードレビューで十分では? わざわざ撮らなくても。**
A: コードレビューはレンダリングを予測します。撮影はレンダリングを観測します。`flex` が中央寄せする「はず」、色のコントラストが足りている「はず」――「はず」はすべてピクセルに関する主張で、ピクセルに関する主張はピクセルを見て初めて検証されます。予測は検証ではありません。

**Q: 静止画で、動画の問題まで掴めるの?**
A: 掴みきれません。1フレームでは動きもタイミングもわかりません。だからtenganは冒頭・中間・終端など複数フレームを抜いて、並べて見ます。それでも、フレームとフレームの間で起きる問題は取りこぼします。そこは正直に、静止画の限界です。

**Q: ログインの要る画面も撮れる?**
A: 冷えた状態では撮れません。撮影はログイン壁にぶつかって、その先には届きません。事前に確立したセッション――保存済みのストレージ状態か、ユーザーが渡すテスト用アカウントか――が要ります。届く範囲を撮って、認証の要る面は設定なしでは届かなかったと報告します。ちなみに認証情報を `.claude/tengan.local.md` に書き込むことは絶対にしません。どこから来るかは記録しても、その値は記録しません。

**Q: じゃあこれ、デザイナーの代わりになる?**
A: なりません。tenganが放つのは網であって、審美眼ではありません。はみ出し、読めない文字、作り忘れた空状態――そういう明らかな破綻を、人間の目より先に拾います。色の趣味やデザインの狙いに及ぶ差は、スクリーンショットを添えてあなたに返します。網はデザイナーを置き換えません。デザイナーが見るより前に、明白な穴を塞ぐだけです。

**Q: 天眼通なら、未来のバグも見えるのでは?**
A: 見えません。見えるのは、いま画面に映っているものだけです。それでも、いままで誰も見ていなかったのですから、大きな進歩です。

千里を見通す目も、まず開かなければ、ただ閉じた目です。
