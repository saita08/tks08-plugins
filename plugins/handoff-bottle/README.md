# handoff-bottle

> 「昨日の続き、やるか」
> 「……で、どこまでやったんだっけ」
> 「なんでこの実装にしたんだっけ」
> 「次、何するつもりだったんだっけ」

昨日の自分は、今日の自分にとって赤の他人です。一晩寝れば、下した決定の理由も、次にやろうとしていたことも、きれいさっぱり蒸発しています。だから今日のセッションは、まず「思い出す」という何も生まない儀式から始まります。毎朝、浜辺で潮の匂いを嗅ぎながら、昨日ここに立っていた誰かの足跡を読み解くわけです。

`handoff-bottle` は、その蒸発の前に状態を瓶に詰めて海へ流すプラグインです。次のセッションが、浜辺でそれを拾います。手紙の宛先は、明日のあなたという名の他人です。

## これは何？

作業が一つのセッションで終わらないとき、切れ目で必ず情報が落ちます。いまどういう状態か、なぜこの道を選んだか、次に何をするつもりだったか。これらは会話の中にしか存在せず、会話が閉じれば潮が引くように消えます。次のセッションは、その消えた情報を波打ち際でゼロから拾い集めることになります。

`handoff-bottle` は、中断のときに「いまの状態 / 下した決定とその理由 / 次の一歩 / 触っていたファイル」を定型で書き出し、栓をして流します。瓶はプロジェクトの中の普通のMarkdownファイルなので、割れませんし、海水も染みません。次のセッションが始まると、最新の一本が「前回からの引き継ぎ」として自動で拾い上げられます。あなたは何も思い出さなくていい。瓶が覚えています。

## 特徴

- **四つの枠で状態を封じる**: いまの状態、下した決定とその理由、次の一歩、触っていたファイル。とりわけ二番目が肝心です。決定の理由は最も早く蒸発し、拾い直すのに最も高くつく漂流物だからです。理由の抜けた決定は、次のセッションを同じ議論の渦へ引き戻します
- **瓶は上書きされず、沈む**: 新しい瓶が上に載り、古い瓶が下へ沈みます。直近3本まで海底に残るので、「一昨日はどうだったか」も少しだけ潜って遡れます
- **拾い手は静かに待つ**: セッション開始時、フックが浜を歩いて瓶を探します。あれば最新の一本を引き継ぎとして注入し、無ければ何も言いません。瓶の無いプロジェクトでは、このプラグインは水平線の向こうにいるのと同じです
- **中身は人間が読める漂流物**: ただのMarkdownなので、あなたは瓶の中を読めるし、間違っていれば手で直せるし、コミットして残せるし、別の人へ流せます

## /resume や自動要約と何が違うのか

Claude Code には組み込みの `/resume` があり、コンテキストが伸びれば自動で要約もされます。それらは優秀な航海術です。では、なぜ瓶なのか。

組み込みの仕組みが運ぶのは、機械のセッション内部状態です。あなたはそれを気軽に読めないし、直せないし、そのツールという船の外へ持ち出せません。同僚に渡すことも、リポジトリという港に残すこともできません。瓶が運ぶのは、人間が読める引き継ぎです。「同じセッションをそのまま再開したい」だけなら `/resume` の方が軽い。瓶が値打ちを持つのは、引き継ぎが**読めて・直せて・セッションや人をまたいで残る**べきときです。両者は競合しません。潜水艦と手紙瓶は、役割が違うだけです。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/handoff-bottle:write` | いまの作業状態を四つの枠で瓶詰めし、`.claude/handoff-bottle.local.md` に流す |
| Skill | `suggest-handoff` | 区切りや離脱の気配を読み、瓶詰めをそっと勧める判断 |
| Hook | `SessionStart` | セッション開始時に浜を探し、瓶があれば最新の一本を引き継ぎとして注入する |

## 使い方

インストールすれば、フックとスキルは黙って潮の番を始めます。普段どおり作業してください。

区切りが来ると、Claude が「瓶に詰めておきますか」と聞いてくることがあります。乗るなら任せてください。自分から詰めたいときは `/handoff-bottle:write` を直接呼んでも構いません。特に残したい文脈があれば、引数で一言添えられます。次にそのプロジェクトでセッションを始めると、最新の瓶が自動で読み込まれます。あなたは「どこまでやったっけ」を口にする必要がありません。

## 置き場所

瓶は `.claude/handoff-bottle.local.md` に流れ着きます。プロジェクトの中なので、コミットするかどうかはあなたの判断です。チームで引き継ぎを共有したいならコミットすればいいし、個人の作業メモとして留めたいなら `.gitignore` に沈めておけばいい。ファイルはただのMarkdownなので、瓶の中身が的外れなら手で直しても構いません。

## よくある質問

**Q: 瓶の中身が間違っていたら？**
A: 手で直してください。ただのMarkdownです。瓶は Claude が会話から状態を読み取って書くので、あなたの頭の中だけにあった判断は入っていません。口に出さなかった決定が瓶に入らないのは、人間の引き継ぎとまったく同じ理屈です。

**Q: トークンや鍵を瓶に入れても大丈夫？**
A: だめです。瓶はプロジェクトの中に置かれ、コミットされうるファイルです。この規律はコマンド側に組み込んであり、瓶は秘密を名前で参照するだけで値は書きません。ただし海に流す前に、栓の締まり具合は一度あなたの目で確かめてください。

**Q: セッションが終わったら自動で詰めてくれない？**
A: 詰めません。セッション終了を機械が確実に見極めるのは難しく、勝手に量産すればプロジェクトが空き瓶だらけになります。だから瓶を詰めるのは、あなたが明示的に指示したときだけ。離脱を言い忘れたまま窓を閉じれば、その日の瓶は残りません。そこは自動化しません。何を残すかの判断は、あなたのものです。

**Q: 半年前の瓶まで全部読み返せる？**
A: 読み返せません。海に残るのは直近3本まで。古い瓶は静かに沈んで消えます。長期の記録が要るなら、それは瓶ではなくコミットログや設計ドキュメントという名の、もっと丈夫な陸地の仕事です。

流した瓶を、次のあなたが必ず拾えるとは限らない。それでも、栓をして流すのは今日のあなたにしかできない。
