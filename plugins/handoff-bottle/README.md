# handoff-bottle

> 「昨日の続き、やるか」
> 「……で、どこまでやったんだっけ」
> 「なんでこの実装にしたんだっけ」
> 「次、何するつもりだったんだっけ」

昨日の自分は、今日の自分にとって赤の他人です。一晩寝れば、下した決定の理由も、次にやろうとしていたことも、きれいさっぱり蒸発しています。そして今日のセッションは、まず「思い出す」という何も生まない作業から始まります。毎回です。

`handoff-bottle` は、その蒸発の前に状態を瓶に詰めて、海に流すプラグインです。次のセッションが、浜辺でそれを拾います。

## これは何？

作業が一つのセッションで終わらないとき、セッションの切れ目で必ず情報が落ちます。いまどういう状態か、なぜこの道を選んだか、次に何をするつもりだったか。これらは会話の中にしか存在せず、会話が閉じれば消えます。次のセッションは、その消えた情報をゼロから組み立て直すことになります。

`handoff-bottle` は、中断のときに「いまの状態 / 下した決定とその理由 / 次の一歩 / 触っていたファイル」を定型で書き出し、瓶詰めにします。瓶はプロジェクトの中の普通のMarkdownファイルです。そして次のセッションが始まると、最新の瓶が「前回からの引き継ぎ」として自動で読み込まれます。あなたは何も思い出さなくていい。瓶が覚えています。

## /resume や自動要約と何が違うのか

Claude Code には組み込みの `/resume` があり、コンテキストが伸びれば自動で要約もされます。それらは優秀です。では、なぜ瓶なのか。

組み込みの仕組みが運ぶのは、機械のセッション内部状態です。あなたはそれを気軽に読めないし、直せないし、そのツールの外へ持ち出せません。同僚に渡すことも、リポジトリに残すこともできません。

瓶が運ぶのは、人間が読める引き継ぎです。ただのMarkdownなので、あなたは中身を読めるし、間違っていたら直せるし、コミットして残せるし、別の人に渡せます。「同じセッションをそのまま再開したい」だけなら組み込みの `/resume` の方が軽い。瓶が値打ちを持つのは、引き継ぎが**読めて・直せて・セッションや人をまたいで残る**べきときです。両者は競合しません。役割が違います。

## 三つの部品

### コマンド — 瓶を詰める

`/handoff-bottle:write` を呼ぶと、いまの作業状態が定型テンプレートで `.claude/handoff-bottle.local.md` に書き出されます。テンプレートは四つの枠でできています。**いまの状態**（何が途中か）、**この作業で下した決定とその理由**（最も蒸発しやすく、最も再構築が高くつく部分）、**次の一歩**（具体的な一つの行動）、**触っていたファイル**（次にどのファイルを開けばいいか）。

瓶は上書きされません。新しい瓶が上に載り、古い瓶が下に沈みます。直近3本まで残るので、「一昨日はどうだったか」も少しだけ遡れます。

### フック — 瓶を拾う

SessionStart フックが、セッション開始時にプロジェクトの中の瓶を探します。あれば、最新の1本を「前回からの引き継ぎ」として注入し、それが引き継ぎであることがわかる一言を添えます。無ければ、何も言いません。瓶が無いプロジェクトでは、このプラグインはいないのと同じです。

### スキル — 瓶を勧める

作業が大きな区切りを迎えたとき、あるいはあなたが「今日はここまで」「続きは明日」と離脱を示唆したとき、Claude が瓶詰めをそっと提案します。押し売りはしません。一度断れば、そのセッションではもう言いません。あなたが乗ったときだけ、コマンドが走ります。

## 使い方

インストールすれば、フックとスキルは黙って働き始めます。普段どおり作業してください。

区切りが来たら、Claude が「瓶に詰めておきますか」と聞いてくることがあります。乗るなら任せてください。自分から詰めたいときは `/handoff-bottle:write` を直接呼んでも構いません。

次にそのプロジェクトでセッションを始めると、最新の瓶が自動で読み込まれます。あなたは「どこまでやったっけ」を言う必要がありません。

## 置き場所

瓶は `.claude/handoff-bottle.local.md` に置かれます。プロジェクトの中なので、コミットするかどうかはあなたの判断です。チームで引き継ぎを共有したいならコミットすればいいし、個人の作業メモとして留めたいなら `.gitignore` に入れておけばいい。ファイルはただのMarkdownなので、瓶の中身が的外れなら手で直しても構いません。

## できないこと・正直な制約

- **瓶の質は、詰めたときの状態の質を超えません。** Claude が会話から状態を読み取って書くので、会話に現れなかった決定は瓶にも入りません。頭の中だけにあった判断は、口に出さないと瓶に入らないのは、人間の引き継ぎと同じです。
- **秘密は瓶に入れてはいけません。** 瓶はプロジェクトの中に置かれ、コミットされうるファイルです。トークンや鍵を扱った作業でも、瓶はそれを名前で参照するだけで、値は書きません。この規律はコマンド側に組み込んでありますが、コミット前に一度自分の目で確かめてください。
- **自動では詰めません。** セッション終了を機械が確実に判定するのは難しく、勝手に瓶を量産すればプロジェクトがノイズだらけになります。だから瓶を詰めるのは、あなたが（あるいは提案に乗って）明示的に指示したときだけです。離脱を言い忘れたまま閉じれば、瓶は残りません。そこは自動化しません。判断はあなたのものです。
- **保持は直近3本まで。** 履歴を全部残すツールではありません。古い瓶は沈んで消えます。長期の記録が要るなら、瓶ではなくコミットログや設計ドキュメントの領分です。
