# embassy

> 「Claude には自分の価値観を丁寧に仕込んだ」
> 「ChatGPT にも同じことを言いたい」
> 「Gemini にも……あれ、同じ内容をまた一から書くのか」
> 「しかも文字数制限で入りきらない」

丹精込めて育てた個人憲法は、`CLAUDE.md` の中に閉じたままでは、Claude と話しているときにしか効きません。`embassy` は、その制度を各サービスの様式に翻訳して送り出す、あなたの外交団です。

## これは何？

自分がどう扱われたいか、どう働いてほしいか。それを言語化した `CLAUDE.md` は、育てるほどに効いてきます。けれどその恩恵は、Claude Code の中で完結してしまう。同じ人間が ChatGPT に指示を書き、Gemini の gem を設定するとき、また一から、しかも各サービスの様式と文字数制限に合わせて書き直すことになります。面倒なので、たいてい適当な短縮版で済ませ、結果としてサービスごとに別人格の Claude や GPT が育ちます。

`embassy` は、あなたの制度文書を「本国」と定め、各国(各AIサービス)へ大使を送り出します。大使の仕事は翻訳です。本国の憲法を、赴任先の様式と作法と文字数制限に合わせて訳し直す。どのAIと話しても、同じ価値観が通じるようにする。それが大使館の役目です。

## 翻訳には規律がある

短くするのは簡単です。原則を「たとえば A、たとえば B」と事例で置き換えれば、たいてい制限に収まります。しかしそれは翻訳ではなく、劣化です。原則の価値は、書かれた事例の外側——著者が想像しなかった状況——にまで届く一般性にあります。事例列挙に堕とした瞬間、読み手はその事例を定義だと受け取り、一般性は死にます。

だから `embassy` の翻訳は、この一線を守ります。短くするときは、価値の言い方を締めることで縮める。それでも入らなければ、原則を骨抜きにして全部残すのではなく、**丸ごと落とす原則を選び、何を落としたかをあなたに告げます**。中身の残った十の原則は、骨を抜かれた二十の原則に勝るからです。数値上の制限で切ったときは、黙って三分の一を捨てて「完成しました」とは言いません。何を国境で降ろしたかを、必ず添えて返します。

## 仕組み

`/embassy:draft <対象サービス>` が翻訳の本体です。対象(ChatGPT のカスタム指示、Gemini の gem、汎用システムプロンプトなど)を指定すると、その様式と制約に合わせて本国の制度文書を訳し、ファイルに書き出します。原本を指定しなければ、あなたの `CLAUDE.md` を本国とみなします。

翻訳結果はファイルに落ちます。それを各サービスの設定欄に貼るのは、あなたの仕事です。`embassy` は外部サービスに何も送信しません。大使が書簡を認めるところまでが仕事で、国境を越えて手渡すのは人間の役目です。

本国の憲法は、時とともに改訂されます。すると各国版は、静かに古くなる。何も繋がっていないので、古くなったことすら気づけない。`/embassy:diff` は、前回翻訳した時点の本国の姿(内容のハッシュ)を憶えていて、今の本国と突き合わせ、どの国版の更新が必要かを報告します。改訂のたびに全部訳し直す必要はありません。ズレた大使館にだけ、訓令を出し直せばいい。

## 使い方

翻訳する。

```
/embassy:draft chatgpt
```

これで `~/.claude/CLAUDE.md` が ChatGPT のカスタム指示の様式に訳され、`embassy/chatgpt.md` に書き出されます。文字数制限で何かを落としたなら、その内訳も報告されます。あとはそのファイルを開いて、ChatGPT の設定に貼るだけです。

原本や対象を変えるなら引数で。

```
/embassy:draft gemini ~/.claude/CLAUDE.md
/embassy:draft system-prompt ./team-constitution.md
```

本国を改訂したあとは、どの国版が古びたかを確認する。

```
/embassy:diff
```

古くなった対象と、それぞれを訳し直すコマンドが返ってきます。

## 出力先と状態の置き場所

- 翻訳結果は既定でカレントディレクトリの `embassy/<対象>.md` に書き出します。出力先はコマンドで指定できます。
- どの原本をどの対象に、いつ、どの内容で訳したかは `~/.claude/embassy/state.json` に記録します。`/embassy:diff` はこれを読んで古さを判定します。この記録はマシン単位——あなたの憲法とその翻訳は、プロジェクトをまたいであなたのものだからです。

## できないこと

- **外部サービスへは何も送りません。** これが `embassy` の固い一線です。翻訳をファイルに書くところまでが仕事で、各サービスへ貼るのは人間がやります。自動送信の便利さと引き換えに、あなたの制度がどこへ流れるか分からなくなる事態を、このプラグインは選びません。
- **知らないサービスの様式は捏造しません。** 対応する対象は同梱の定義にあります。未対応の対象を指定されたら、対応一覧を出して尋ねます。当て推量の制約は必ず間違っていて、それに合わせた翻訳は国境で弾かれます。
- **原則を事例列挙に短縮しません。** 制限に収めるためでも、一般性は売り渡しません。収まらないなら原則を丸ごと落とし、そう告げます。
- **本国の履歴は持ちません。** `embassy` が憶えているのは「前回訳した時点のハッシュ」だけです。原本の版管理は、原本のある場所の git の仕事です。

## 前提

- `shasum` か `sha256sum` が使えること(原本の内容ハッシュを取るのに使います。macOS・Linux の標準にあります)。
- 翻訳したい制度文書があること。既定は `CLAUDE.md` ですが、任意のファイルを本国にできます。
