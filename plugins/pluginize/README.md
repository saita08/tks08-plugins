# pluginize

> 「これ、前にもやったな」
> 「同じ手順、今週3回目だ」
> 「プラグインにすれば一生やらなくて済むのにな」
> 「……それはそれとして、今日もやらない」

人類はこうして、未来の自分を毎日見殺しにしています。`pluginize` は、その現場に居合わせて「いま、やりません?」と肩を叩いてくる、おせっかいなプラグインです。

## これは何？

作業をしていると、再利用できる構造がぽろぽろ生まれます。繰り返しの手順、いつの間にか固まった段取り、頭の中にしかない謎のコツ。どれも本来ならプラグインにして、次からは指一本で済ませるべきものです。

問題は、それに「気づく」のも面倒で、「作る」のはもっと面倒で、結局その知見は今日のセッションと一緒に蒸発する、という点にあります。あなたは悪くありません。プラグイン化は、いつだって「あとでやる」の墓場行きなのです。

`pluginize` は、その墓場の前に立って通行人に声をかける係です。あなたが作業を一区切りつけるたび、後ろで静かに「いま、何か再利用できるものが生まれませんでした?」と見ています。何も無ければ、何も言いません。本当に何も言いません。空気が読める方のプラグインです。候補を見つけたときだけ、一度だけ、控えめに袖を引きます。

そしてあなたが「やる」と言ったら、プラグインを作る面倒な作業は**全部このセッションの外**でやります。今の会話は、一滴も汚れません。

## 三つの層

このプラグインは三つの層でできていて、それぞれが一つの誓いを立てています。

### 観測 — 広く見て、しかし割り込まない

Stop hook が、あなたのターンが終わるたびに裏で軽く `claude -p`（働き者の Haiku）を走らせ、再利用できる構造の候補を探します。判断軸は三つ。**再現性**（また同じことやるでしょ?）、**手続きの結晶度**（その手順、もう順番まで決まってますよね?）、**暗黙知**（それ、どこにも書いてない秘伝のタレでは?）。どれか一つでも刺されば候補です。

検出は広く見ます。見逃した候補は二度と戻ってこないからです。ただし拾うのは、軸が作業の中に実際に顔を出したときだけ。「ありそう」で拾い始めると提案が壁紙になり、あなたが読まなくなった瞬間、本物の候補まで一緒に沈むからです。既存のコマンドやスキルが回した作業も拾いません。あれはもうプラグインになっている構造の残像であって、候補ではないのです。提案は1セッションに1回まで。一度スルーされた候補が、次の区切りでまた袖を引いてきたりはしません。そして候補が無いときは、完全に黙ります。喋るのが例外、黙るのが日常。寡黙なタイプです。

### 提案・承認 — 効果を添えて、決して勝手に動かない

候補を見つけたら、ただ「これどうですか」と置いていくのではなく、「プラグインにすると、こういう得がありますよ」とご利益まで添えてお出しします。押し売りではありません。あなたが「よし」と言うまで、生成は一文字も始まりません。

この「よし」には二つの意味が乗っています。一つは**あなたが本当にそれを欲しいか**の確認。もう一つは、このプラグインが**勝手に暴走しない無害な奴である**ことの証明です。お行儀の良さを、説明文ではなく振る舞いで示します。ついでに、過去の「よし」は引き継ぎません。毎回ちゃんと聞き直します。一度頷いたからといって、なし崩しに次々作り始めたりはしない、という固い約束です。

### 隔離生成 — 重い荷物は、ここに持ち込まない

承認が出たら、`claude --bg` で隔離されたセッションを起動し、そこで [`/plugin-dev:create-plugin`](https://code.claude.com/docs/en/plugins) を走らせます。プラグインの正しい作り方を知っているのはそっちの専門家なので、`pluginize` は素直に丸投げします。自分でやろうとしないのが、できる仲介者の条件です。

メインセッションの仕事は、候補をまとめて専門家に渡すところまで。あとは別室の話です。その別室は agent view から覗いて会話できるので、プラグインの細かい仕上げはそちらで詰めてください。`claude --bg` のファイル編集は worktree に隔離され、完成品はあなたのローカルに、そっと着地します。

## 使い方

インストールすれば、観測層は黙って働き始めます。あなたは普段どおり作業を続けてください。何もする必要はありません。むしろ何もしないのが正しい使い方です。

候補が見つかると、ターンの区切りで一度だけ、提案がそっと差し込まれます。乗り気なら `/pluginize:propose-plugin`。承認ゲートをくぐって、生成セッションが立ち上がります。気が乗らなければ、無視してください。傷つきません。プラグインですから。

hook を待たず、自分から「これプラグインにできる?」と `/pluginize:propose-plugin` を呼んでも構いません。せっかちな方向けの入口です。

## 生成物はローカルに着地する。公開は勧めるだけ

できあがったプラグインは、まずあなたのローカルに着地します。どれだけ汎用的に見えても、例外はありません。ローカルは試運転の場であり、公開は取り消しのきかない片道切符であり、そして「これは万人向けだ」という判定は、しれっと外れることもあるからです。

もし汎用的に見えたら、生成ワークフローは最後に「公開、考えてみては?」と**勧めて**きます。あくまで勧めるだけで、勝手にボタンは押しません。マーケットプレースがあるのか、公開したいのか、どうやるのか――そこから先は、全部あなたの領分です。世界に何を出すかを決めるのは、いつだってあなたです。

## 構成

```
pluginize/
├── .claude-plugin/
│   └── plugin.json
├── hooks/
│   ├── hooks.json          # Stop hook（asyncRewake で裏で観測、候補時のみ rewake）
│   └── detect.sh           # transcript を claude -p に渡し3軸で候補判定。再帰ガード付き
├── skills/
│   └── propose-plugin/
│       ├── SKILL.md         # 承認 → claude --bg で生成セッションを起動
│       └── references/
│           ├── detection-axes.md    # 何をプラグイン化すべきか
│           ├── plugin-value.md      # プラグイン化すると何が得られるか
│           ├── constitution.md      # 各層を貫く価値と判断の枠組み
│           └── generation-prompt.md # 生成セッションに渡すプロンプトの雛形
└── README.md
```

## 前提

- `claude` CLI がパスにあること。観測（`claude -p`）も生成（`claude --bg`）も、これ一本で回します。API キーは不要。Claude Code の認証に相乗りします。
- `/plugin-dev:create-plugin` を提供するプラグイン（`plugin-dev`）が使えること。生成はまるごとこの専門家に任せるので、いないと提案だけして手詰まりになります。

## FAQ

**毎ターン裏で `claude -p` が走るって、メインのコンテキストが汚れない?**
汚れません。観測は `asyncRewake` で非同期に走るので、あなたのターンを待たせません。候補が無ければ一文字も出力せず沈黙するので、メインには何も入ってきません。喋るのは候補があるときの一行だけ。それ以外は、いないのと同じです。

**ユーザースコープでインストールしたら、私の `settings.json` が上書きされちゃう?**
されません。プラグイン同梱の hooks は、あなたの hooks とメモリ上で仲良くマージされて並列で走ります。あなたの設定には指一本触れません。

**`claude -p` を hook から呼んだら、それ自身の Stop hook がまた発火して無限ループしない?**
しません。`detect.sh` は再帰ガードを持っています。`claude -p` を呼ぶときに合言葉（環境変数）を立てておき、子セッションの hook はそれを見て「あ、これは身内だ」と即座に引き返します。ウロボロスは未然に防がれます。
