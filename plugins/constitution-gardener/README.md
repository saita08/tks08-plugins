# constitution-gardener

> 「そこはタブじゃなくてスペースって言ったよね」
> 「あー、はいはい、コミットメッセージに絵文字入れないでって前も……」
> 「聞いてる? この口調やめてって、何回目?」

同じ訂正を、あなたは何度もしています。相手は忘れるからではありません。どこにも書いていないからです。書いていないことは、次のセッションで蒸発します。あなたは毎回、同じことを、口で言い直しています。

`constitution-gardener` は、その繰り返しを「憲法への欠陥報告」として拾い上げる庭師です。あなたが Claude に二度以上くり返した同じ訂正を、あなたのトランスクリプトの中から見つけ出し、「これ、`CLAUDE.md` に書いておきませんか」と、原則の文案まで添えて差し出します。

## これは何？

あなたには憲法があります。個人の `~/.claude/CLAUDE.md`、プロジェクトごとの `CLAUDE.md`。そこに書いた原則は、次のセッションでも効きます。書いていない原則は、あなたの口の中にしかありません。

一度の訂正は、ただの訂正です。二度目の同じ訂正は、違います。それは「この規律は憲法に載るべきだったのに載っていない」という、明確な欠陥報告です。人間は同じことを三度言わされてやっと「これ書いておくか」と思いますが、その「やっと」までの間、同じ摩擦が延々と燃え続けます。

庭師の仕事は、その欠陥報告を拾って、原則の形に整えて、あなたの前に置くことです。ただそれだけです。書き込みも、コミットも、PR も、庭師はしません。**判断と批准は、いつでもあなたのものです。** 庭師は文案を起草するだけ。載せるか、直すか、捨てるかを決めるのは、あなたです。憲法の書き手はあなたであって、庭師ではないからです。

## 何を「摩擦」と見なすのか

庭師が探すのは、あなたのトランスクリプトに残る三種類の足あとです。いずれも、直前の Claude の行動とセットで拾われます。何を訂正したのかだけでなく、何に対する訂正だったのかまで見ないと、原則には昇華できないからです。

- **訂正** — 「そうじゃなくて」「違う、こうして」。Claude のやり方を、あなたが上書きした瞬間。
- **却下** — 「それはやめて」「元に戻して」。Claude の提案や変更を、あなたが押し返した瞬間。
- **言い直し** — 同じ依頼を、あなたが角度を変えて言い直した瞬間。一度で伝わらなかった、という記録。

そして肝心なのは、**一度きりの摩擦は候補にしない**ことです。プロジェクト固有の事情かもしれないし、その日限りの気分かもしれない。二度以上、別の機会に、同じ類の摩擦が起きたときにだけ、庭師はそれを「原則に昇華すべき何か」として拾います。繰り返しこそが、それが偶然でなく規律であることの証拠だからです。

## 使い方

区切りのついたときに、`/constitution-gardener:harvest` を実行してください。庭師は前回の続きから最近のセッションを掘り、繰り返された摩擦を探し、候補ごとにレポートを返します。各候補には、こう並びます。

1. **観測された事例** — いつ、どのセッションで、何が起きたか。引用つきで。憶測ではなく、実際に起きたことの記録です。
2. **失敗の類の命名** — その摩擦は「どういう類の失敗」なのか。個別の事件ではなく、次に別の形で現れる同じ穴に名前をつけます。
3. **宛先の判定** — この原則は、あなた個人の `~/.claude/CLAUDE.md` に載るべきか、それとも特定プロジェクトの `CLAUDE.md` に載るべきか。あらゆるプロジェクトで効くべき規律と、この現場だけの約束は、家が違います。
4. **改正文案** — 実際に憲法に貼れる、原則としての文章。事例の列挙ではなく、値と理由として書かれた原則です。

あとはあなたが読んで、決めます。庭師との対話の中で文案を練り直してもいいし、そのまま自分で `CLAUDE.md` に貼ってもいい。庭師は貼りません。

## 前回どこまで掘ったかを憶えている

トランスクリプトは膨大です。数百メガバイトに達することもあります。毎回すべてを読み直すのは、時間の無駄であり、同じ候補を何度も出すことにもなります。

そこで庭師は、`~/.claude/constitution-gardener/state.json` に「前回どこまで処理したか」のカーソルを持ちます。次の `harvest` は、そのカーソル以降の新しいセッションだけを掘ります。一度のharvestで掘るセッション数には上限があるので、履歴が積み上がっていても、一回の実行が暴走することはありません。

## 何も見つからないのは、良い知らせ

大量の繰り返し訂正が「見つからない」ことは、失敗ではありません。むしろ逆です。それは、あなたの憲法がすでに機能していて、同じ摩擦が二度と起きていない証拠です。

庭師は、0件のときにそれを正しく祝います。「候補なし」は、掘り損ねたのではなく、庭がよく手入れされている、ということです。うるさく何かをひねり出したりはしません。畑に雑草がなければ、庭師は静かに帰ります。

## できないこと・正直な制約

- **庭師は書きません。** `CLAUDE.md` への追記も、コミットも、PR も一切しません。案を出すまでが庭師の仕事で、載せるかどうかは常にあなたとの対話の中で決まります。これは制約であると同時に、庭師が勝手にあなたの憲法を書き換えない、という約束でもあります。
- **摩擦の検出は完璧ではありません。** トランスクリプトから「これは訂正だ」と読み取るのは判断を伴う作業で、皮肉や軽口を訂正と誤読したり、丁寧な言い直しを見逃したりします。だから候補は候補であって、決定ではありません。最後にふるいにかけるのは、あなたの判断です。
- **一度のharvestが見るのはカーソル以降だけです。** 大昔のセッションに埋もれた繰り返しは、初回のharvestが届く範囲の外にあれば拾えません。掘る範囲は「最近」に寄せてあります。
- **抽出には `claude` CLI を使います。** 摩擦の読み取りは軽量なサブエージェント（`claude -p`）に分割して行うので、`claude` がパスにあることが前提です。API キーは要りません。Claude Code の認証に相乗りします。

## 構成

```
constitution-gardener/
├── .claude-plugin/
│   └── plugin.json
├── commands/
│   └── harvest.md          # カーソル以降のセッションを掘り、繰り返された摩擦を候補化してレポート
├── skills/
│   └── draft-principle/
│       ├── SKILL.md         # 摩擦を原則に昇華する文案の書き方
│       └── references/
│           ├── principle-writing.md   # 原則として書く規律（事例列挙でなく値として）
│           └── friction-taxonomy.md   # 摩擦の三類型と、宛先制度の判定基準
└── README.md
```
