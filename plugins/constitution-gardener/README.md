# constitution-gardener

> 「そこはタブじゃなくてスペースって言ったよね」
> 「あー、はいはい、コミットメッセージに絵文字入れないでって前も……」
> 「聞いてる? この口調やめてって、何回目?」

同じ雑草を、あなたは毎朝抜いています。抜いても抜いても、翌朝また生えている。相手が忘れっぽいからではありません。根を残したまま地上部だけ刈っているからです。書いていない規律は、セッションが終わるたびに地表から消え、次の朝きっちり同じ場所から芽を出します。

`constitution-gardener` は、その生え際に根を張る庭師です。あなたが二度以上くり返した同じ訂正を、あなた自身のトランスクリプトという土壌から掘り当て、「これ、`CLAUDE.md` の花壇に植えておきませんか」と、原則の苗まで仕立てて差し出します。

## これは何？

あなたには憲法があります。個人の `~/.claude/CLAUDE.md` という自宅の庭、プロジェクトごとの `CLAUDE.md` という現場の花壇。そこに植えた原則は、越冬します。次のセッションでも根づいたまま芽吹く。植えていない原則は、あなたの口の中にしかなくて、毎朝そこから言い直すたびに、また一年草のように枯れます。

一度の訂正は、ただの落ち葉です。踏んで通り過ぎればいい。二度目の同じ訂正は、落ち葉ではありません。それは「この規律は花壇に植わっているべきだったのに植わっていない」という、地面からの明確な欠陥報告です。人間は同じことを三度言わされてやっと「植えておくか」と思いますが、その「やっと」までの間、同じ根が毎朝あなたの手を煩わせ続けます。庭師は、三度目を待ちません。二度目の芽で、もう根の在り処を見抜きます。

## 何を「雑草」と見なすのか

庭師が地面を這って探すのは、三種類の足あとです。いずれも、直前の Claude の行動とセットで掘り起こされます。何を抜いたのかだけでなく、何の根から生えた芽だったのかまで見ないと、苗には仕立てられないからです。

- **訂正** — 「そうじゃなくて」「違う、こうして」。Claude のやり方を、あなたが上から植え替えた瞬間。
- **却下** — 「それはやめて」「元に戻して」。Claude の提案を、あなたが土ごと押し戻した瞬間。
- **言い直し** — 同じ依頼を、あなたが角度を変えてもう一度撒いた瞬間。一度目の種が芽を出さなかった、という記録。

そして肝心なのは、**一本きりの芽は苗にしない**ことです。プロジェクト固有の土壌かもしれないし、その日限りの風で飛んできた種かもしれない。二度以上、別の朝に、同じ根から芽が出たときにだけ、庭師はそれを「花壇に植え替えるべき何か」として掘り上げます。同じ場所から二度生えることこそ、それが偶然の飛来ではなく根を張った多年生であることの、動かぬ証拠だからです。

## 使い方

区切りのついたときに、`/constitution-gardener:harvest` を実行してください。庭師は前回鍬を入れたところから最近のセッションを掘り返し、二度以上くり返された芽を選り分け、候補ごとにレポートを返します。各候補には、こう並びます。

1. **観測された事例** — いつ、どのセッションで、何が生えたか。引用つきで。庭師は根も見ずに苗を仕立てたりはしません。掘って、確かめて、土のついたまま見せます。
2. **失敗の類の命名** — その芽は「どういう根」から出たのか。個別の一本ではなく、次に別の場所から出る同じ根に名前をつけます。名前のない根は、また見失うからです。
3. **宛先の判定** — この苗は、あなた個人の `~/.claude/CLAUDE.md` という自宅の庭に植えるべきか、特定プロジェクトの `CLAUDE.md` という現場の花壇に植えるべきか。どこでも育つべき多年草と、この土壌でしか意味を持たない植栽は、植える場所が違います。
4. **改正文案** — そのまま花壇に植えられる、原則としての苗。事例を並べただけの寄せ植えではなく、値と理由で一本に仕立てた原則です。

あとはあなたが手に取って、決めます。庭師との対話の中で苗を仕立て直してもいいし、そのまま自分で `CLAUDE.md` に植えてもいい。**土に植える手は、いつでもあなたの手です。** 庭師は苗を差し出すところで手を止めます。花壇の主はあなたであって、庭師ではないからです。

## 前回どこまで掘ったかを憶えている

トランスクリプトは、耕せば耕すほど広くなる畑です。数百メガバイトに達することもあります。毎朝畑を端から端まで掘り返すのは、時間の無駄であり、昨日掘った場所から同じ芽を何度も引き抜くことにもなります。

そこで庭師は、`~/.claude/constitution-gardener/state.json` に「前回どこまで鍬を入れたか」の目印を刺しておきます。次の `harvest` は、その目印から先の、まだ手をつけていない畝だけを掘ります。一度の収穫で掘る畝の数には上限があるので、畑がどれだけ広がっても、一回の収穫で庭師が息を切らして倒れることはありません。

## 畑に雑草がなければ、庭師は静かに帰る

大量のくり返し訂正が「見つからない」ことは、不作ではありません。むしろ豊作です。それは、あなたの憲法という花壇がすでによく手入れされていて、同じ雑草が二度と根づいていない証拠です。

庭師は、0件のときにそれを正しく祝います。「候補なし」は、掘り損ねたのではなく、庭がよく整っている、ということです。何もないところから無理やり雑草をひねり出して「ほら仕事しました」と見せたりはしません。抜くべき草がなければ、庭師は道具をしまい、帽子を軽く上げて、静かに畑を後にします。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/constitution-gardener:harvest` | 目印から先のセッションを掘り、二度以上くり返された摩擦を苗にしてレポート |
| Skill | `draft-principle` | 摩擦を原則の苗に仕立てる作法。事例の寄せ植えではなく、値と理由で一本に |

Hook は使いません。庭師はあなたが `harvest` と呼んだときにだけ畑に出ます。呼んでいないのに勝手に他人の花壇をいじる庭師は、庭師ではなく不法侵入です。

## よくある質問

**Q: 庭師が勝手に `CLAUDE.md` を書き換えたりしませんか？**
A: しません。追記も、コミットも、PR も、庭師は一切しません。苗を仕立てて差し出すところが庭師の仕事の果てで、植えるかどうかは常にあなたとの対話で決まります。これは制約であると同時に、他人の花壇に無断で植える庭師を雇わずに済む、という約束でもあります。

**Q: 皮肉や軽口を「訂正」と読み違えませんか？**
A: 読み違えます。土から生えたものが花か雑草かを見分けるのは判断の伴う仕事で、庭師も時々、まだ蕾の花を雑草と間違えて掘り、丁寧な言い直しを見逃します。だから候補は苗であって、植栽の決定ではありません。最後にふるいにかけるのは、あなたの目です。

**Q: 大昔のあの繰り返しも掘り当ててくれますか？**
A: 一度の `harvest` が鍬を入れるのは目印から先の畝だけです。畑の遠い隅、初回の鍬が届く範囲の外に埋もれた根は、そのままそこに残ります。掘る範囲は意図して「最近」に寄せてあります。庭師は畑全体を毎朝掘り返す代わりに、少しずつ、確実に前へ進みます。

**Q: この庭師、動くのに何が要りますか？**
A: 摩擦の読み取りは軽量なサブエージェント（`claude -p`）に分けて行うので、`claude` がパスにあることが前提です。API キーは要りません。Claude Code の認証に相乗りします。読むのはすべてローカルの `~/.claude/` 配下で、外の畑へは土ひとかけら持ち出しません。

---

良い庭は、雑草がないことではなく、抜いた雑草が二度と同じ場所から生えてこないことで測られます。庭師は、その「二度と」を花壇に植えに来ます。
