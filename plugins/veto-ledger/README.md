# veto-ledger

> 「その案は無しで」
> 「承知しました!」
> （三日後、別のセッションにて)
> 「こういうのはどうでしょう、名前を変えた例のあれ」
> 「……それ、この前ボツにしたやつだよね?」

一度断ったはずの提案が、少し衣を変えて戻ってくる。これは、ユーザーの信頼を最も静かに、最も確実に削っていく失敗のひとつです。断るのにも労力を使ったのに、また同じものが出てくる。`veto-ledger` は、却下された提案を丁重に「供養」し、成仏させて二度と化けて出ないようにする台帳です。

## これは何？

Claude は毎セッションまっさらな状態で始まります。昨日あなたが「その方針は無し」と却下したことを、今日の Claude は知りません。だから、悪気なく、同じ方向を——ときには少し見た目を変えて——また提案してきます。

`veto-ledger` は、却下された方向と「なぜ却下されたか」を台帳(`.claude/veto-ledger.local.md`)に記録します。そして新しいセッションが始まるたびに、その台帳を「過去に却下された方向、再提案しないこと」として Claude に渡します。一度断った「無し」が、以後のセッションでも「無し」であり続けます。

肝は、記録するのが**個別の事例ではなく、却下の軸(原則)**だという点です。「PRを2つに分ける案を却下した」とだけ書けば、その一件しか止まりません。「このプロジェクトは1機能1PRを崩さない」と書けば、まだ誰も思いついていない同種の提案まで、まとめて成仏します。

## 仕組み

三つの部品でできています。

### 検知 — 却下に気づいて、しかし勝手には書かない

会話の中であなたが提案を却下したり方針を禁じたりすると(「それはやめて」「その方針は無し」「却下」)、Claude はそれを台帳に記録する候補として拾います。このとき Claude は、却下された方向そのものだけでなく、「なぜダメなのか」という軸を言葉にしようとします。そして「これを供養帳に記録しておきますか?」と、書こうとしている一行を見せて、控えめに確認します。あなたが頷いてから記録されます。

なぜ確認するのか。この台帳は次以降の全セッションの提案を縛ります。軸が間違っていたり広すぎたりすると、本来は歓迎すべき良い提案まで、静かに封じてしまうからです。だから台帳に載るのは「あなたが載せると言ったもの」だけです。

### 注入 — 台帳があるときだけ、静かに渡す

セッションが始まると、`veto-ledger` はプロジェクトに台帳があるかを見ます。あれば、その中身を「再提案しないこと」として Claude の文脈に渡します。無ければ何もしません。台帳が育ちすぎたときは直近のものだけを渡すので、文脈が却下履歴で埋まることもありません。

### 一覧と手動記録

`/veto-ledger:list` で、これまでに却下された方向と軸を一覧できます。会話を待たず自分で記録したいときは `/veto-ledger:add <却下された方向> | <却下の軸>`。呼び出したこと自体が承認なので、追加の確認は挟みません。

## 使い方

インストールすれば、注入層は黙って働き始めます。あなたが提案を却下すると、Claude が「供養しておきますか?」と控えめに聞いてくるので、良ければ頷いてください。次のセッションから、その方向は再提案されなくなります。

過去の却下を眺めたいときは `/veto-ledger:list`。自分で足したいときは `/veto-ledger:add`。台帳そのものは `.claude/veto-ledger.local.md` に置かれているので、直接開いて直すこともできます。

## 台帳はプロジェクトの中に置かれる

台帳は `.claude/veto-ledger.local.md`、つまりあなたのプロジェクトの中に住みます。Claude だけが読む私的メモではありません。却下という判断は、あなたとチームの見える場所に残るべきだからです。チームで使うならバージョン管理に入れれば、「これはやらない」という合意ごと共有できます。手元だけに留めたいなら `.gitignore` へ。

## 設定

- `VETO_LEDGER_MAX_ENTRIES` — 一度のセッションに注入する却下項目の最大件数(デフォルト 30)。台帳が大きくなったとき、直近この件数だけを渡します。

## できないこと・正直な制約

- **却下を自動では判定しません。** 「いま提案が却下された」と気づくのは会話の流れを読む Claude の判断であって、機械的な検知ではありません。曖昧な難色は拾われないことがあります。確実に残したいときは `/veto-ledger:add` を使ってください。
- **軸の質は、その場の判断に依存します。** 良い台帳は「個別事例でなく原則」で書かれますが、この一般化は Claude の判断で行われます。軸が狭すぎれば似た提案を取りこぼし、広すぎれば良い提案まで封じます。`/veto-ledger:list` でときどき見直し、必要なら手で直してください。
- **台帳は注入するだけで、強制はしません。** 渡された却下履歴を Claude がどれだけ忠実に守るかは、その場の判断に委ねられます。強い示唆であって、機械的なブロックではありません。
- **台帳が肥大すると、古い却下は注入から外れます。** 上限を超えた分は直近優先で切られます。全件を常に効かせたいなら `VETO_LEDGER_MAX_ENTRIES` を上げてください。
- **`jq` が必要です。** 注入は `jq` で文脈を組み立てます。無い環境では注入は黙ってスキップされます。
