# veto-ledger

> 「その案は無しで」
> 「承知しました!」
> （三日後、別のセッションにて)
> 「こういうのはどうでしょう、名前を変えた例のあれ」
> 「……それ、この前ボツにしたやつだよね?」

一度断ったはずの提案が、少し衣を変えて戻ってくる。これは、ユーザーの信頼を最も静かに、最も確実に削っていく失敗のひとつです。断るのにも労力を使ったのに、また同じものが現世に舞い戻る。`veto-ledger` は、却下された提案を丁重に供養し、成仏させて二度と化けて出ないようにする台帳です。この台帳が営むのは、迷える案を送り出す弔いの仕事です。

## これは何？

Claude は毎セッションまっさらな状態で生まれ変わります。前世であなたが「その方針は無し」と却下したことを、今生の Claude は覚えていません。だから、悪気なく、同じ方向を——ときには少し見た目を変えて——また差し出してきます。未練を残した案は、名前を変え、装いを変え、しつこく枕元に立ちます。

`veto-ledger` は、却下された方向と「なぜ却下されたか」を台帳(`.claude/veto-ledger.local.md`)に戒名のごとく記します。そして新しいセッションが始まるたびに、その台帳を「過去に却下された方向、再提案しないこと」として Claude に渡します。一度手を合わせた「無し」が、以後のセッションでも「無し」であり続けます。

肝は、供養するのが個別の亡霊ではなく、却下の軸(原則)だという点です。「PRを2つに分ける案を却下した」とだけ記せば、その一体しか鎮まりません。「このプロジェクトは1機能1PRを崩さない」と記せば、まだ誰も思いついていない同種の亡霊まで、まとめて彼岸へ渡ります。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Skill | `veto-capture` | 会話の中の却下を拾い、方向とその軸を言葉にして台帳への記録を提案する |
| Command | `/veto-ledger:add` | 却下を待たず、`<却下された方向> \| <却下の軸>` の形で直接供養する |
| Command | `/veto-ledger:list` | これまでに供養した方向と軸を一覧する |
| Hook | SessionStart | 台帳があれば、その中身を再提案禁止としてセッション冒頭に注入する |

## 仕組み

三つの部品でできています。

### 検知 — 却下に気づいて、しかし勝手には書かない

会話の中であなたが提案を却下したり方針を禁じたりすると(「それはやめて」「その方針は無し」「却下」)、Claude はそれを台帳に記録する候補として拾います。このとき Claude は、却下された方向そのものだけでなく、「なぜダメなのか」という軸を言葉にしようとします。そして「これを供養帳に記録しておきますか?」と、記そうとしている一行を見せて、控えめに手を合わせます。あなたが頷いてから記帳されます。

なぜ確認するのか。この台帳は次以降の全セッションの提案を縛ります。軸が間違っていたり広すぎたりすると、本来なら迎えるべき良い案まで、静かに封じてしまうからです。生きている案を早まって葬るのが、供養係の最も犯してはならない過ちです。だから台帳に載るのは「あなたが載せると言ったもの」だけです。

### 注入 — 台帳があるときだけ、静かに渡す

セッションが始まると、`veto-ledger` はプロジェクトに台帳があるかを見ます。あれば、その中身を「再提案しないこと」として Claude の文脈に渡します。無ければ何もしません。台帳が育ちすぎたときは直近のものだけを渡すので、文脈が過去帳で埋まることもありません。

### 一覧と手動供養

`/veto-ledger:list` で、これまでに送り出した方向と軸を一覧できます。会話を待たず自分で記帳したいときは `/veto-ledger:add <却下された方向> | <却下の軸>`。呼び出したこと自体が承認なので、追加の確認は挟みません。

## 使い方

インストールすれば、注入層は黙って働き始めます。あなたが提案を却下すると、Claude が「供養しておきますか?」と控えめに聞いてくるので、良ければ頷いてください。次のセッションから、その方向は再提案されなくなります。

過去の却下を眺めたいときは `/veto-ledger:list`。自分で足したいときは `/veto-ledger:add`。台帳そのものは `.claude/veto-ledger.local.md` に置かれているので、直接開いて直すこともできます。

## 台帳はプロジェクトの中に置かれる

台帳は `.claude/veto-ledger.local.md`、つまりあなたのプロジェクトの中に住みます。Claude だけが読む私的メモではありません。却下という判断は、あなたとチームの見える場所に残るべきだからです。チームで使うならバージョン管理に入れれば、「これはやらない」という合意ごと共有できます。手元だけに留めたいなら `.gitignore` へ。

## 設定

- `VETO_LEDGER_MAX_ENTRIES` — 一度のセッションに注入する却下項目の最大件数(デフォルト 30)。台帳が大きくなったとき、直近この件数だけを渡します。

## よくある質問

**Q: 却下した瞬間に、自動で供養してくれますか?**
A: 「いま提案が却下された」と気づくのは会話の流れを読む Claude の判断であって、機械的な検知ではありません。曖昧な難色は拾われないことがあります。はっきり成仏させたいときは `/veto-ledger:add` で戒名を授けてください。生返事のような却下では、亡霊は往生できません。

**Q: 軸って、ちゃんと書いてくれるんですか?**
A: 良い台帳は「個別事例でなく原則」で書かれますが、この一般化は Claude の判断で行われます。軸が狭すぎれば似た亡霊を取りこぼし、広すぎれば良い案まで封じます。`/veto-ledger:list` でときどき過去帳を検め、必要なら手で直してください。読経の質は、たまに施主が確かめるに越したことはありません。

**Q: 台帳に載せたのに、また同じ案が化けて出ました。**
A: 台帳は注入するだけで、Claude を機械的にブロックはしません。渡された過去帳をどれだけ忠実に守るかは、その場の判断に委ねられます。強い示唆であって、結界ではありません。それでも、供養なしに毎回一から祓うよりは、ずっと静かな夜になります。

**Q: 古い却下が効かなくなりました。**
A: 台帳が上限を超えると、古い却下は直近優先で注入から外れます。全件を常に効かせたいなら `VETO_LEDGER_MAX_ENTRIES` を上げてください。過去帳が分厚くなるほど、古い戒名は棚の奥へ下がっていきます。

**Q: `jq` が要ると聞きました。**
A: 注入は `jq` で文脈を組み立てます。無い環境では注入は黙ってスキップされます。会話を壊してまで読経を続けるほど、この寺は不作法ではありません。

却下とは、可能性を一つ手放す決断です。手放したものが夜ごと戻ってこないよう、この台帳は今日も一体、静かに送り出します。南無。
