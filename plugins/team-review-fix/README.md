# team-review-fix

> "レビュー指摘が5件来ました"
> "了解、直します"
> （1時間後）
> "3番と5番、同じファイルだったのに別々に読み込んで二重課税されました"
> "あと2番の修正、それ対症療法では？"
> "...最初からちゃんと分担すればよかった"
>
> はい。このプラグインはその「最初からちゃんと」を自動でやります。

## これは何？

レビュー指摘を貼り付けると、**即席チームを結成して並列で直してくれる**プラグインです。

あなたがやることは指摘を貼り付けるだけ。あとは Claude がフェロー（指揮役）に変貌し、ファイル単位でチームメイトを召喚して仕事を割り振ります。フェローは自分では一行もコードを書きません。偉い人はそういうものです。

各チームメイトは調査・計画・実装を一気通貫で担当。「計画だけ立てて実装は別の人に」みたいな伝言ゲームはしません。コードを読んだ人間が直す。当たり前のことを当たり前にやります。

そして計画が対症療法的なら容赦なく差し戻します。「try-catch で握りつぶす」？ 却下。「as any で黙らせる」？ 論外。このフェロー、厳しいです。

## 使い方

テキストで直接:
```
/team-review-fix エラーハンドリングが不十分です（src/api.ts:30）、命名がtypoしています（src/utils.ts:15）
```

ファイルで渡す（myrule-review の出力ファイルをそのまま食わせるとか）:
```
/team-review-fix notes/code-review-pr123.md
```

何も考えずに:
```
/team-review-fix
```
聞いてくれます。優しいのはここだけです。

## 裏で何が起きてるの？

1. **指摘を解析** — テキストやファイルから指摘事項を抽出
2. **ファイル単位でグルーピング** — 同じファイルへの指摘はまとめる。ここが肝。3番と5番が同じファイルなら、同じチームメイトが両方やる
3. **横断的な方針決定** — 「エラーハンドリングのパターン統一しよう」みたいな、先に決めないとチームメイトがバラバラになる判断を片付ける
4. **チーム結成** — ファイルごとにチームメイトを召喚。3ファイルなら3人。10ファイルなら10人。民主主義ではなく実力主義
5. **調査と計画** — 各チームメイトが事実確認し、修正計画を立てる。「本当にバグか？」から始める慎重派
6. **計画レビュー** — フェローが計画をチェック。根本解決じゃなければ「やり直し」の一言
7. **実装** — 承認された計画だけ実装。修正ごとにコミット。未承認の計画を勝手に実装したら大問題
8. **完了報告** — 全チームメイトの結果をまとめて報告。お疲れさまでした

## なぜファイル単位？

指摘番号で分けると、Agent A が `src/api.ts` を読み、Agent B も `src/api.ts` を読み、同じファイルを2回読んだ上に編集が衝突する、という地獄絵図になります。

ファイル単位で分ければ:
- 1回の読み込みで複数指摘を処理できる（エコ）
- 変更の競合が原理的に起きない（平和）
- 担当が明確（責任の所在）

## 対症療法チェックって？

フェローは計画段階で「それ、根本解決ですか？」と問います。

こういうのは却下されます:
- エラーが出るから `try-catch` で握りつぶす（症状を消しただけ）
- 型が合わないから `as any` を付ける（型システムへの冒涜）
- null が来るから `if (x != null)` を追加する（null が来ること自体がおかしいのでは？）
- バグの説明コメントを追加する（コメントはコードではない）

こういうのは承認されます:
- null が来る原因を上流で修正する
- 型定義を正しく直す
- バリデーションをシステム境界に追加する

厳しいですが、あとで困るのはあなたです。フェローは困りません。AIなので。

## FAQ

**Q: チームメイトは何人作られるの？**
A: 変更対象ファイルの数だけ。3ファイルなら3人。100ファイルなら... レビュー指摘が100ファイルに及ぶなら、まずPRの粒度を見直してください。

**Q: 横断的な設計判断って具体的に何？**
A: 「AbortController のパターンを統一しよう」「エラーメッセージのフォーマットを揃えよう」みたいなやつです。先に決めないと、チームメイトAが `throw new Error()` を書いてる横でチームメイトBが `return { error: ... }` を書きます。

**Q: フェローは本当に一行もコードを書かないの？**
A: はい。Constitutional AI で禁止されています。憲法違反です。弾劾されます。

**Q: 対症療法的かどうか、本当に正しく判定できるの？**
A: 完璧ではありません。でも「try-catch で握りつぶす」レベルのものは確実に弾きます。微妙なケースはフェローの判断に委ねられます。Claude の判断力を信じてください。たぶん。

**Q: myrule-review と組み合わせられる？**
A: むしろそれが想定された使い方です。`/myrule-review` で指摘を出力 → そのファイルを `/team-review-fix` に食わせる。レビューから修正まで全自動。人間は承認ボタンを押すだけ。...承認ボタンすら押しません。フェローが押します。
