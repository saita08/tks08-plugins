# paste-tube

> 「あっちのセッションでこう言われたんだけど」（ぺたっ）
> 「この前のエラーログ、貼っとくね」（ぺたっ）
> 「別プロジェクトのレビュー結果です」（ぺたっ）
> ――三ターン後――
> 「……これ、どこから持ってきたやつだっけ？」

昔のデパートには気送管というものがありました。会計伝票と釣り銭をカプセルに詰め、圧縮空気で「シュポン」と屋上の集中計算室へ飛ばす。管を伝う音が店内に響き、しばらくすると釣り銭が同じ管を通って返ってきました。`paste-tube` は、セッションとセッションの間にその管を一本、通すプラグインです。

## これは何？

あなたは日々、手でコンテンツをセッション間に運んでいます。片方の窓からレビュー結果をコピーして、別の窓に貼る。エラーログをプロジェクトをまたいで持ち込む。「あっちのセッションでこう言われた」を伝える。売り場から売り場へ、伝票を手で走らせているわけです。

このとき、二つのものが失われます。一つは**出典**。貼った直後はわかっていても、会話が三ターン進めば、そのログが本番のものだったか検証環境のものだったか、このプロジェクトのものか隣の売り場のものか、誰も覚えていません。もう一つは**存在そのもの**。会話が流れれば貼った内容は消え、また元の売り場まで取りに戻ることになります。`paste-tube` は貼り込みをカプセルに封じ、出典と日付を印字して管に通します。別のセッションは、そのカプセルを出典つきで受け取れます。シュポン、と隣の部屋へ。

## 特徴

- **中身はそのまま封じる**: 要約も言い換えもしません。受け取る側が見るべきは、実際に言われた原文だからです。伝票を清書し直したら、それはもう伝票ではありません
- **出典が印字される**: カプセルは frontmatter に出典プロジェクトと日付を持ちます。これがただの貼り込みとの分かれ目です。どの売り場から、いつ飛ばされた一枚なのかが、開けた瞬間にわかります
- **受け取っても管に残る**: カプセルは受け取りでは消えません。何度でも、どのセッションからでも受け取れます。指が滑って受け取っても、原本が壊れることはありません
- **飛ばすのは、あなたが入れたときだけ**: 全部の貼り込みを自動でアーカイブする管も作れましたが、採りませんでした。使い捨ての伝票まで全部集中室へ送れば、本当に運びたい一枚がその山に埋もれ、`~/.claude/` が静かに肥り続けます。何を運ぶ価値があるかは、機械ではなくあなたの判断です

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/paste-tube:send <ラベル>` | 直前の会話内容（または指定内容）を、出典つきカプセルとして `~/.claude/paste-tube/` に封じる |
| Command | `/paste-tube:receive` | 保管カプセルを新しい順に一覧し、選んだものを出典つきで会話へ取り込む |
| Skill | `carry-between-sessions` | 貼り込みが「出典を失い、かつこのターンを越えて生きる」と読んだとき、管の利用をそっと案内する判断 |

## 使い方

運びたいものがあれば `/paste-tube:send <ラベル>`。ラベルは必須です。後でカプセルの山から見つけ出すための、伝票の表書きになります。別のセッションで受け取りたければ `/paste-tube:receive` を呼ぶと、保管されたカプセルがラベル・出典プロジェクト・日付の一行で新しい順に並びます。選んだものが「これは○○プロジェクトから、○月○日に封じられたレビューです」と枠つきで会話に降りてきます。どの世界から来た内容なのかを、見失いません。

加えて、あなたが「これを別セッションに持っていきたい」「あっちでこう言われた」といった受け渡しの気配を出したとき、Claude が気送管の利用を案内します。

## 置き場所

カプセルは `~/.claude/paste-tube/` に着きます。マシン単位の保管庫、いわば屋上の集中計算室なので、そのマシンのどのプロジェクト・どのセッションからも受け取れます。ファイル名は日付とラベルでできていて、新しい順に並びます。中身はただのMarkdownなので、エディタで開いて読むこともできます。

## よくある質問

**Q: カプセルは別のマシンにも飛ぶ？**
A: 飛びません。この管はネットワークの向こうへは通っていません。カプセルはローカルのファイルで、`paste-tube` は外部へ何も送信しません。別のマシンへ運びたいなら、それはあなたの手で管の外を運ぶ作業で、このプラグインの領分ではありません。

**Q: 貼ったものは自動で保存される？**
A: されません。上に書いたとおり、これは設計上の判断です。明示的に `send` したものだけが管を通り、封じ忘れた貼り込みは普通に会話とともに流れて消えます。全部を自動で送れば、本当に運びたい一枚が使い捨ての山に埋もれるからです。

**Q: 認証情報が混ざったログを封じても大丈夫？**
A: 用心してください。カプセルは `~/.claude/` に永続化され、ディスクからの削除は完全な消去を保証しません。運ぶ内容に認証情報が含まれていれば封じる前に Claude が知らせますが、最終的に何を管へ入れるかは、あなたの判断です。

**Q: 受け取ったカプセルは消える？**
A: 消えません。受け取りは消費ではないので、原本は管に残り続けます。掃除はあなたが明示的にファイルを削除して行います。裏を返せば、放っておくとカプセルは屋上に貯まります。ときどき `~/.claude/paste-tube/` を覗いて、要らない一枚を捨ててください。

シュポン、と音がして、隣の部屋の誰かが、あなたの封じた一枚を出典ごと受け取る。
