# paste-tube

> 「あっちのセッションでこう言われたんだけど」（ぺたっ）
> 「この前のエラーログ、貼っとくね」（ぺたっ）
> 「別プロジェクトのレビュー結果です」（ぺたっ）
> ――三ターン後――
> 「……これ、どこから持ってきたやつだっけ？」

昔の建物には気送管というものがありました。カプセルに書類を詰めて、圧縮空気で別の部屋にひゅっと飛ばす。`paste-tube` は、セッションとセッションの間にその管を通すプラグインです。

## これは何？

あなたは日々、手でコンテンツをセッション間に運んでいます。片方の窓からレビュー結果をコピーして、別の窓に貼る。エラーログをプロジェクトをまたいで持ち込む。「あっちのセッションでこう言われた」を伝える。

このとき、二つのことが失われます。一つは**出典**。貼った直後はわかっていても、会話が三ターン進めば、そのログが本番のものだったか検証環境のものだったか、このプロジェクトのものか隣のものか、誰も覚えていません。もう一つは**存在そのもの**。会話が流れれば貼った内容は消え、また元の場所から貼り直すことになります。

`paste-tube` は、貼り込みをカプセルに封じます。カプセルには出典（どのプロジェクトから、いつ）が記録され、ディスクに残ります。別のセッションは、そのカプセルを出典つきで受け取れます。管を通って、書類がひゅっと隣の部屋へ。

## 二つのコマンド

### 送る — カプセルに封じる

`/paste-tube:send <ラベル>` で、直前の会話内容（あるいは指定した内容）をカプセルとして `~/.claude/paste-tube/` に保管します。カプセルは frontmatter に出典プロジェクトと日付を持つMarkdownファイルです。中身は**そのまま**封じます。要約も言い換えもしません。受け取る側が見るべきは、実際に言われた原文だからです。

ラベルは必須です。後でカプセルの山から見つけ出すための名前になります。

### 受け取る — カプセルを開ける

`/paste-tube:receive` で、保管されたカプセルの一覧が新しい順に出ます。それぞれ、ラベル・出典プロジェクト・日付が一行で見えます。選んだものが会話に取り込まれます――**出典つきで**。「これは○○プロジェクトから、○月○日に封じられたレビューです」と枠がついて出るので、どの世界から来た内容なのかを見失いません。ただの貼り込みとの違いは、まさにここです。

受け取ってもカプセルは消えません。何度でも、どのセッションからでも受け取れます。要らなくなったら、あなたがファイルを消してください。

## なぜ自動アーカイブが無いのか

`paste-tube` は、あなたの貼り込みを自動で保存したりしません。UserPromptSubmit フックで全部の貼り込みを片っ端からアーカイブする、という設計もありえました。採りませんでした。

理由は二つ。**ノイズ**――貼り込みの大半はその場で使って終わるもので、それを全部カプセル化すれば、本当に運びたい一枚が無数の使い捨てに埋もれます。カプセルの一覧が使い物にならなくなれば、プラグインごと使われなくなります。そして**容量**――全貼り込みの自動保存は、`~/.claude/` を静かに肥らせ続けます。

だから `paste-tube` が動くのは、あなたが明示的に `send` / `receive` したときだけです。何を運ぶ価値があるかの判断は、機械ではなくあなたのものです。管は、あなたがカプセルを入れたときにだけ空気を送ります。

## 使い方

インストールすれば、二つのコマンドが使えます。加えて、あなたが「これを別セッションに持っていきたい」「あっちでこう言われた」といった受け渡しの文脈を出したとき、Claude が気送管の利用をそっと案内します。

運びたいものがあれば `/paste-tube:send <ラベル>`。別のセッションで受け取りたければ `/paste-tube:receive`。それだけです。

## 置き場所

カプセルは `~/.claude/paste-tube/` に置かれます。マシン単位の保管庫なので、そのマシンのどのプロジェクト・どのセッションからも受け取れます。ファイル名は日付とラベルでできていて、新しい順に並びます。中身はただのMarkdownなので、エディタで開いて読むこともできます。

## できないこと・正直な制約

- **外部には一切送りません。** カプセルはローカルのファイルです。`paste-tube` はネットワークの向こうへ何も送信しません。別のマシンへ運びたいなら、それはあなたの手でやる作業で、このプラグインの領分ではありません。
- **自動では保存しません。** 上に書いたとおり、これは設計上の判断です。明示的に `send` したものだけが残ります。封じ忘れた貼り込みは、普通に会話とともに流れて消えます。
- **秘密の扱いに注意。** カプセルは `~/.claude/` に永続化されます。ディスクからの削除は完全な消去を保証しません。運ぶ内容に認証情報が含まれていれば、封じる前に Claude が知らせますが、最終的に何を残すかはあなたの判断です。
- **受け取りは消費ではありません。** 受け取ってもカプセルは残り続けます。掃除は明示的な削除で、あなたが行います。裏を返せば、放っておくとカプセルは貯まります。ときどき `~/.claude/paste-tube/` を覗いて、要らないものを捨ててください。
