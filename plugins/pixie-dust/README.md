# pixie-dust

> 「SVGでマスコット描いて」
> 「できました」
> 「……なんか、のぺっとしてない?」

AIにSVGキャラクターを描かせた人は、全員この「のぺっと感」に会っています。正円の頭、左右対称の目、linearで平行移動するジャンプ。間違ってはいない。ただ、生きていない。`pixie-dust` は、Claudeが最初の `<path>` を書く前に振りかける妖精の粉です。

## これは何？

ピーターパンの伝承では、妖精の粉を振りかけられたものは飛べるようになります。このプラグインが振りかけるのは、描かれたものが生きて見えるための原則です。

のぺっと感の正体は、生成モデルの性質そのものにあります。モデルは訓練分布の最頻値——数学的に最も安い形——へ回帰します。正円は楕円より安く、対称は非対称より安く、`linear` はどんな cubic-bezier よりも安い。その安さの合計が、あの生気のなさです。逆に言えば、作られた質感は最頻値からの意図的な逸脱に宿ります。逸脱は観測可能なルールに落とせるので、生成の前に読ませれば、出力は変わります。

## 何が変わるのか

のぺっと感は三つの因子でできています。粉はその全部に効きます。

- **形**: キャラクターの本体に `<circle>` や `<ellipse>` を使うことをやめ、制御点をずらした非対称の有機的なベジェパスで描きます。目や耳の位置も数ピクセル単位で崩します。意識には上らないが、「テンプレートだ」と脳が読む閾値の下には潜り込む、その程度のずれです
- **陰影と影**: 単色ベタ塗りをやめ、静止画の品質が要る場面では中心を外した radialGradient と feTurbulence で立体と質感を与えます。接地影は「ベタ塗りの灰色楕円」をやめ、接地点の濃いコアと透明へ減衰する柔らかい外周の二層で組み、色は純グレーではなく光源とパレットに合わせて設計します。光源はひとつに決め、ハイライトと影の向きを一致させます
- **動き**: 最大の因子です。跳ぶ前に沈み（予備動作）、空中で伸びて着地で潰れ（squash & stretch）、耳と尻尾は一拍遅れて揺れ戻り（follow-through）、目標を行き過ぎてから戻り（オーバーシュート）、二軸の移動は直線でなく弧を描き（arcs）、影は高度に合わせて縮み（環境との連動）、まばたきは本体と異なる周期で重なります（生体ノイズ）。イージングは `ease` や `linear` という既製の言葉を使わず、物理イベントに合わせて cubic-bezier で設計します

原則は本プラグインの発明ではありません。Disneyの「アニメーションの12原則」のうちWeb/SVGに最も強く翻訳される部分と、Material Design や IBM Carbon といったモーションデザインシステムの定説を、SVG生成の現場に届く形に束ねたものです。同じ形を、この原則のあり・なしで並べると効果は一目でわかります。動きのパターンは「イベント冒頭の10〜20%で逆方向へ」のような比率の言葉でスキルに収められていて、検証済みの実数値は導出のための一例として添えられています。完成品のサンプルは同梱していません。具体例は書けば書くほど、次の生成をその具体例へ引きずるからです。

効くのはキャラクターだけではありません。スピナーは回転に呼吸（線長の伸縮）を重ね、複数要素の登場はスタガーで振り付け、装飾ブロブは不均一な変位で組み、ロゴは線描リビールに follow-through を添える——非キャラクターのSVGも、同じ最頻値回帰に同じ処方が効きます。

## 使い方

何もしなくていいです。それがこのプラグインの設計です。

「SVGでキャラクター描いて」「マスコット作って」「ローディングアニメーションが欲しい」——SVGで図やキャラクターや動きを作る仕事が始まった瞬間に、スキルが自動で発火し、最初のパスが書かれる前に原則が適用されます。スラッシュコマンドはありません。粉は、振りかけるものであって、命令するものではないので。

## よくある質問

**Q: SVGなら何でも生き返る?**
A: いいえ。SVGにはボーンもメッシュ変形もないので、塗られた面の内側を滑らかに歪ませる表現——頬の潰れ、呼吸の波打ち、房ごとの髪揺れ——はパーツ単位の transform で擬似するのが天井です。その水準が製品の核なら、スキルはコードで粘らず、Rive や WebGL への分岐を明示的に提案します。天井の名前を言えることも、粉の効能のうちです。

**Q: 動きだけの話?**
A: 動きが最大の因子ですが、形と陰影の原則は静止画にもそのまま適用されます。動かないマスコットも、正円と左右対称をやめた瞬間に顔つきが変わります。

**Q: これでデザイナーになれる?**
A: なれません。粉が消すのは「機械が作った」という署名であって、趣味の良さを注入するわけではありません。ただ、署名が消えるだけで、見られる時間は変わります。

粉は一振りで足ります。振り忘れたときだけ、あの、のぺっとした顔がこちらを見ています。
