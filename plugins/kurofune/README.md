# kurofune

> 沖に見慣れぬ船影が四つ。
> 「攘夷だ！打ち払え！」
> 「お待ちください。あれは、コードが書けます」
> 「……上陸を許可する。ただし奉行の検分つきだ」

嘉永六年、黒船は日本を開国させました。このプラグインは逆をやります。xAI の Grok Build という異国の蒸気船を、あなたの港に迎え入れて、**仕事を頼む**のです。Claude Code は奉行所として設計と検分を握ったまま、力仕事は艦隊に発注する。外圧は、条約さえ正しく結べば、腕のいい取引相手です。

かつては奉行の手代（監督エージェント）が黒船とあなたを仲介していました。いまは違います。奉行本人——**メインの Claude 会話**——が桟橋に立ち、手信号で艦隊を動かします。間に通訳を挟むほど、報告は薄くなり、sessionId は港の霧に消えます。当奉行所はそれを禁じます。

## これは何？

`kurofune` は、Grok Build CLI をヘッドレスで走らせて、Claude Code のメイン会話から**サブエージェントのように**使える道具にします。プラグインを入れると、桟橋に三本の旗が立ちます。

| 旗 | 意味 |
| --- | --- |
| `task` | 新しい船を呼ぶ |
| `resume` | 同じ船と交渉を続ける |
| `doctor` | 港の点検（船体と手形） |

これは Docker でも、月額のかかる沖合サーバでもありません。Claude Code があなたの机の上で小さな伝令を起こし、stdin/stdout で用を足して解散させる——そういう**地元の使い走り**です。金がかかるのは、黒船そのものを動かした分だけです。既定の船種は **grok-4.5**（`KUROFUNE_MODEL` や旗の `model` / `-m` で差し替え可）。

肝心なのは、黒船が**話の続きを覚えている**ことです。Grok のセッションは港（`~/.grok/sessions`）に係留されていて、最初の派遣が返す `sessionId` を持っていれば、「さっきのあれ、デフォルト引数を付けろ」という雑な追撃が通じます。毎回まっさらな船に一から事情を説明する開国交渉ではなく、同じ使節と交渉を続けるのです。修正のたびに新しい船を呼ぶのは、コンテキストの再建費を毎回払う攘夷運動であり、当奉行所はこれを禁じます。

成果の正本は艦長の自己申告ではなく **作業ツリーの差分** です。旗が戻す報告には `gitStatus` と `gitDiffStat` が載るので、奉行は桟橋のその場で検分できます。生の航海日誌は `resultFile`（既定: `$TMPDIR/kurofune-results/`）にも残るので、伝言が途中で切れても `sessionId` だけは港に残ります。

並列もできます。ただし艦隊を並べるのは機構ではなく**段取り**です。奉行が仕事を互いに触れないファイル領域へ切り分け、一領域につき一隻。二つの領域が切り分けられないなら、それは二つの任務ではなく一つの任務です。統合と検分は、いつも通り奉行所が行います。

## 通商条約 五箇条

黒船は強力ですが、外国船です。だから上陸の前に条約を結びます。飾りではありません。ヘッドレスの艦上には TTY がなく、write モードの黒船は自分の砲門を自分で許可します（`--always-approve`）。その力を安全にする方法は、条文だけです。

**一、入港は git の港に限る。** 黒船が書き込んでよいのは git 管理下のディレクトリだけです。すべての変更が `git diff` で検分でき、一言で差し戻せる水域でしか、外国船は自由に動けません。

**二、荷揚げは桟橋まで。commit はさせない。** 変更は作業ツリーに置いて帰らせます。何が歴史に入るかの判断は、奉行所とあなたのものです。任務書には毎回「Do not commit」と明記されます。

**三、積荷に機密を載せない。** 任務書に書いたものは海を渡ります。秘密は船に載せず、認証情報の置き場を黒船に教えることもしません。

**四、艦長の自己申告は証拠にならない。** 「できました」は検分ではありません。奉行は `git diff` を読み、変更されたファイルを開き、テストがあれば走らせます。観測したものだけが報告に載ります。二、三度の是正で収束しなければ、粘らず「失敗」と正直に報告します。

**五、見学のみの上陸もある。** レビューモード（`review: true` / `-r`）では書き込みとシェルを道具の水準で外します。読むだけの任務はこちらで。`stopReason: Cancelled` は「もう一回」ではなく「この任務は write モードか、もっと狭い仕様が必要だった」という信号です。

## 使い方

会話で言うだけです。プラグイン導入後に Claude Code を一度再起動すると、旗が手札に並びます。

```
このリファクタ、Grokに投げて
マイグレーションを三つの領域に割って、kurofune を並列で走らせて
さっきのセッションでテスト X が落ちてるから直させて
```

裏では全てが一本の桟橋を通ります。奉行（メイン会話）が直接旗を振るのが正道です。手代エージェントに丸投げして戻ってこない報告を待つのは、旧条約です。

```
kurofune.sh task   [-r] [-C DIR] [-m MODEL] "任務書"          # 新しい船を呼ぶ
kurofune.sh resume [-r] [-C DIR] [-m MODEL] SESSION_ID "追撃"  # 同じ船と交渉を続ける
kurofune.sh doctor                                            # 港の点検
```

## 開国の手続き

蒸気船の調達と通行手形は、あなた自身の手で行います。プラグインは点検はしますが、あなたの環境に勝手に手を入れません。

1. `curl -fsSL https://x.ai/cli/install.sh | bash` で Grok Build を据え付ける
2. `grok login` で手形を得る（SuperGrok または X Premium+ の勘定が要ります。黒船は無料では来ません）
3. 心配なら `doctor` の旗、または `kurofune.sh doctor` — 船体（バイナリ）と手形（認証キャッシュ）を確かめて正直に申告します

| 変数 | 船渠での意味 |
| --- | --- |
| `KUROFUNE_GROK_BIN` | 船体の置き場（既定 `~/.grok/bin/grok`） |
| `KUROFUNE_MODEL` | 既定の船種（既定 `grok-4.5`） |
| `KUROFUNE_RESULT_DIR` | 航海日誌の控えを置く棚 |
| `KUROFUNE_TIMEOUT_MS` | 旗が戻るのを待つ上限（既定 30 分） |

## 旧条約（レガシー）

`kurofune-worker` という手代エージェントは、旗（MCP）が使えない港のための予備です。旗が立っている港では、奉行が自ら振ってください。手代に任せたまま霧が出ると、黒船は帰ってきているのに報告だけが届かない、という哀しい開国劇になります。
