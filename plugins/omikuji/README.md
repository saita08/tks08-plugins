# omikuji

> 朝、エディタを開く。コーヒーはまだ来ていない。
> 「今日はどんな一日になるかな」
> 神社なら、ここでおみくじを引く。
> 開発には、その儀式が無い。

当社へようこそ。ここは開発の朝に一枚のみくじを授ける社です。一日の始まりに、今日の吉凶を引いて背筋を伸ばす、あの小さな儀式を、エディタの前に持ち込みます。ただし、当社のみくじ箱には仕掛けがあります。中に入っているのは番号を書いた竹札ではなく、**昨日までのあなた自身の実際の利用データ**です。占いの形をした、振り返り。それがここで授かる一枚の正体です。

## これは何？

普通のおみくじは、箱を振って出た番号で決まります。`omikuji` の箱の中には、あなたの `stats-cache.json` と `history.jsonl` が納められています。直近あなたがどう働いたか、稼働のリズム、休みの有無、深夜帯の重み、ツール呼び出しの跳ね方を読んで、今日の運勢を一枚に仕立てます。

だから、出てくる運勢には必ず**根拠**があります。「凶」が出ても、それは呪いではありません。「直近は毎日ぶっ通しで、昨日のツール呼び出しは普段の3倍でした。今日は少し休みなさい」という、データからの託宣です。凶を引いて祟られるのではなく、凶を引くほど働いたあなたを、社が心配しているのです。「大吉」が出れば、それは直近のリズムが健やかで持続可能だった、という記録の裏づけがあります。運勢の脇には、それを導いた実際の数字が必ず添えられます。

引けるのは、一日一回。おみくじは二度引いたらおみくじではない、という古来の作法に従います。凶が出たからといって、大吉が出るまで引き直せる社は、社ではなく自動販売機です。

## 一枚のみくじに書かれること

`/omikuji:draw` を引くと、こんな一枚が授かります。

- **運勢** — 大吉・中吉・小吉・末吉・凶のいずれか。直近の稼働リズムが健やかで持続可能なら吉方向へ、無理が見えれば戒めの方向へ傾きます。乱数ではなく、あなたの直近の働き方が決めます。
- **ラッキーコマンド** — あなたが**持っているのに、最近使っていない**スラッシュコマンドから一つ。履歴に一度は現れた、間違いなくあなたのものです。「最後に使ったのは12日前」という最終使用日が根拠として添えられます。持っていないコマンドは勧めません。あなたのものでない幸運を授けるのは、luck の顔をした嘘だからです。ちなみに `/Users/...` のようなファイルパスは、先頭が斜線でもコマンドとは数えません。パスを幸運として授ける社は、その一点で信用を失います。
- **本日の戒め** — 直近の実際のパターンから導いた、一文の警句。「昨日はエージェント起動が目立ちました。本日の凶方位は多重召喚」のように、必ず実データの根拠を名指します。根拠のない戒めは、このプラグインが避けたい「ただの乱数」そのものです。
- **末尾の一行** — このみくじが何日ぶんの記録に基づいたかを、そっと記します。占いではなく振り返りだった、という受領証です。授かった一枚が、賽銭箱の底からではなく、あなた自身の台帳から出てきたことの証です。

## 儀式にフックは無い

このプラグインには、フックがありません。セッションが始まったら勝手に引く、ということはしません。

朝のおみくじが儀式になるのは、**自分の手で引く**からです。社に詣で、賽銭を入れ、箱を振る。あの一連の動作があるから、出た一枚に意味が宿ります。参道を歩かず、境内に入った瞬間にみくじが宙から降ってきたら、それはもう授与ではなく投げつけです。自動で目の前に差し出されたおみくじは、占いではなく通知です。だから `omikuji` は、あなたが `/omikuji:draw` と打つのを待ちます。引きたい朝に、引きたい人が引く。それが儀式の条件です。

## 白紙のみくじ

まだ利用の記録が無いとき、新規インストール直後や、統計を無効化しているとき、このおみくじは運勢を占いません。

その代わり、**白紙のみくじ**を返します。「まだ占うべき記録がありません。ここで運勢を出せば、それはただの作り話です」と正直に告げて、記録が貯まってからの再訪を勧めます。杯が空のときに「空です」と言える神主のほうが、いつ詣でても必ず答えを持っている神主より、信用できます。後者は占っているのではなく、ただ何か言っているだけだからです。

## できないこと・正直な制約

- **記録が無ければ占えません。** データが無いときは白紙のみくじを返します。無いものからは占いません。作り話は、みくじの体裁を借りた嘘です。
- **「今日」はローカルの暦日で数えます。** 一日一回の制限は、あなたのマシンのローカル日付を基準にします。朝の儀式はローカルな「今日」の話だからです。
- **統計には時差があります。** `stats-cache.json` はある時点までの集計です。「直近」はデータが実際に届いている最新の日を指し、それが文字どおり「昨日」とは限りません。みくじはその点を正直に「直近の稼働」と表現します。届いていない昨日を、届いたふりで占ったりはしません。
- **運勢の吉凶は、健やかさの指標であって予言ではありません。** データが示すのは過去の働き方であり、未来ではありません。「凶」は休息の助言、「大吉」は良いリズムの肯定であって、今日の成否を約束するものではありません。社は、あなたの過去を鏡に映すだけで、未来を売りはしません。
- **ラッキーコマンドは履歴にあるものだけ。** あなたが一度も使っていないコマンドは勧めません。おみくじの luck も、あなた自身のデータの範囲を出ません。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/omikuji:draw` | みくじを引く入口。1日1回のガードを確認してから授ける |
| Skill | `draw-omikuji` | ガード確認 → データ読取 → 三部の導出 → 表示 → 記録の本体。根拠のない一枚は決して授けない |

フックは、意図的に置いていません。上記「儀式にフックは無い」のとおりです。

## よくある質問

**Q: 凶が出ました。引き直せますか？**
A: 引き直せません。一日一回です。もう一度引きたくなった時は `~/.claude/omikuji/last-draw` が「本日はもう引いています。今朝のみくじは凶でした」と読み返して、丁重にお断りします。ちなみにその凶は、あなたが働きすぎているという記録からの助言です。引き直すより、休むほうが効きます。

**Q: 大吉が出るようにデータを盛れませんか？**
A: 盛れます。あなたが健やかに働けば、健やかなデータが記録され、みくじは自然に吉へ傾きます。それが唯一の「盛り方」です。データを直接いじる方法はお教えしません。それは賽銭箱に自分で当たりくじを仕込むのと同じで、出た一枚が何の意味も持たなくなります。

**Q: なぜ乱数で決めないのですか？ そのほうが手軽です。**
A: 手軽だからです。乱数のおみくじなら、根拠も、振り返りも、あなた自身も要りません。ただの数字が吉凶を吐くだけです。当社がそれをやらないのは、根拠のある一枚だけが、翌朝もう一度引きたくなる一枚だからです。

**Q: 引いても statusline やどこかが書き換わったりしませんか？**
A: しません。授かるのは画面に映る一枚と、`~/.claude/omikuji/last-draw` への「今日は引いた」という記録だけです。それ以外、あなたのマシンには何も書きません。

## 前提

- `~/.claude/stats-cache.json` および `~/.claude/history.jsonl` が読めること。無ければ白紙のみくじを返すだけで、壊れはしません。
- `python3` がパスにあること。履歴の集計に使います。

一枚引いたら、社のことは忘れて、良い一日を。みくじは、あなたを背中で押すためだけにあります。
