# omikuji

> 朝、エディタを開く。コーヒーはまだ来ていない。
> 「今日はどんな一日になるかな」
> 神社なら、ここでおみくじを引く。
> 開発には、その儀式が無い。

一日の始まりに、小さな儀式が欲しくなることがあります。今日の吉凶を一枚引いて、それをきっかけに背筋を伸ばす、あの感じです。`omikuji` は、その儀式を開発の朝に持ち込みます。ただし、ただの乱数の占いではありません。

このおみくじは、**昨日までのあなた自身の実際の利用データ**から今日の運勢を導きます。占いの形をした、振り返りです。

## これは何？

普通のおみくじは、箱を振って出た番号で決まります。`omikuji` の箱の中には、あなたの `stats-cache.json` と `history.jsonl` が入っています。直近あなたがどう働いたか――稼働のリズム、休みの有無、深夜帯の重み、ツール呼び出しの跳ね方――を読んで、今日の運勢を一枚に仕立てます。

だから、出てくる運勢には必ず**根拠**があります。「凶」が出たら、それは呪いではなく、「直近は毎日ぶっ通しで、昨日のツール呼び出しは普段の3倍でした。今日は少し休みなさい」というデータからの助言です。「大吉」が出たら、それは直近のリズムが健やかで持続可能だった、という記録の裏づけがあります。運勢の脇には、それを導いた実際の数字が必ず添えられます。

引けるのは、一日一回。おみくじは二度引いたらおみくじではない、という古来の作法に従います。

## 一枚のみくじに書かれること

`/omikuji:draw` を引くと、こんな一枚が出ます。

- **運勢** — 大吉・中吉・小吉・末吉・凶のいずれか。直近の稼働リズムが健やかで持続可能なら吉方向へ、無理が見えれば戒めの方向へ傾きます。乱数ではなく、あなたの直近の働き方が決めます。
- **ラッキーコマンド** — あなたが**持っているのに、最近使っていない**スラッシュコマンドから一つ。履歴に一度は現れた、間違いなくあなたのものです。「最後に使ったのは12日前」という最終使用日が根拠として添えられます。持っていないコマンドは勧めません。それは luck の顔をした嘘だからです。
- **本日の戒め** — 直近の実際のパターンから導いた、一文の警句。「エージェント起動が多かった翌日は、本日の凶方位: 多重召喚」のように、必ず実データの根拠を名指します。根拠のない戒めは、このプラグインが避けたい「ただの乱数」そのものです。
- **末尾の一行** — このみくじが何日ぶんの記録に基づいたかを、そっと記します。占いではなく振り返りだった、という受領証です。

## 儀式にフックは無い

このプラグインには、フックがありません。セッションが始まったら勝手に引く、ということはしません。

朝のおみくじが儀式になるのは、**自分の手で引く**からです。神社で、賽銭を入れ、箱を振る。あの一連の動作があるから、出た一枚に意味が宿ります。自動で目の前に差し出されたおみくじは、もう占いではなく通知です。だから `omikuji` は、あなたが `/omikuji:draw` と打つのを待ちます。引きたい朝に、引きたい人が引く。それが儀式の条件です。

## 白紙のみくじ

まだ利用の記録が無いとき――新規インストール直後や、統計を無効化しているとき――このおみくじは運勢を占いません。

その代わり、**白紙のみくじ**を返します。「まだ占うべき記録がありません。ここで運勢を出せば、それはただの作り話です」と正直に告げて、記録が貯まってからの再訪を勧めます。杯が空のときに空だと言える占い師のほうが、いつでも答えを持っている占い師より信用できる、という考え方です。

## できないこと・正直な制約

- **記録が無ければ占えません。** 上記のとおり、データが無いときは白紙のみくじを返します。無いものからは占いません。
- **「今日」はローカルの暦日で数えます。** 一日一回の制限は、あなたのマシンのローカル日付を基準にします。朝の儀式はローカルな「今日」の話だからです。
- **統計には時差があります。** `stats-cache.json` はある時点までの集計です。「直近」はデータが実際に届いている最新の日を指し、それが文字どおり「昨日」とは限りません。みくじはその点を正直に「直近の稼働」と表現します。
- **運勢の吉凶は、健やかさの指標であって予言ではありません。** データが示すのは過去の働き方であり、未来ではありません。「凶」は休息の助言、「大吉」は良いリズムの肯定であって、今日の成否を約束するものではありません。
- **ラッキーコマンドは履歴にあるものだけ。** あなたが一度も使っていないコマンドは勧めません。おみくじの luck も、あなた自身のデータの範囲を出ません。

## 構成

```
omikuji/
├── .claude-plugin/
│   └── plugin.json
├── commands/
│   └── draw.md              # /omikuji:draw — みくじを引く入口
├── skills/
│   └── draw-omikuji/
│       ├── SKILL.md          # ガード確認 → データ読取 → 三部の導出 → 表示 → 記録
│       └── references/
│           ├── fortune-logic.md   # 運勢・ラッキーコマンド・戒めの導出と白紙みくじ
│           └── presentation.md    # みくじの様式と1日1回ガード
└── README.md
```

## 前提

- `~/.claude/stats-cache.json` および `~/.claude/history.jsonl` が読めること。無ければ白紙のみくじを返すだけで、壊れはしません。
- `python3` がパスにあること。履歴の集計に使います。
