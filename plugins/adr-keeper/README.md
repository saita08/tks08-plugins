# adr-keeper

> 「このプロジェクト、1機能1PRでいこう」
> 「了解です」
> ――数日後、別のセッションにて――
> 「この3つの変更、まとめて1つのPRにしておきますね」
> 「!!!」

先週きっちり決めたはずの方針を、今週のClaudeが悪びれもせず逆走する。悪気はありません。決めた瞬間の会話は、そのセッションと一緒に蒸発していて、今のClaudeはそれを一度も見ていないのです。`adr-keeper` は、決定が蒸発する前に瓶に詰めて、次のセッションが必ず開ける棚に置いておく係です。

## これは何？

会話の中では、いろいろな方向性が決まります。「この層がこの責務を持つ」「このフォーマットで統一する」「速さより設定の簡潔さを取る」。どれもその場ではちゃんと理由があって決まっている。ところがその理由はチャットログの中にしか無く、次のセッションのClaudeはログを見ていないので、平気で逆の提案をしてきます。決定が「決定として残っていない」せいで、同じ議論が何度も蒸し返され、方針がふらつくのです。

`adr-keeper` は、この蒸発を止めます。方向性の決定が下されたその場で ADR(Architecture Decision Record)として起票し、以後のすべてのセッションが参照できる棚――`docs/adr/`――に置きます。ADR は「何を決めたか」だけでなく「なぜそう決めたか」を書き残すので、未来のセッションは決定をなぞるだけでなく、その理由まで受け継ぎます。理由が分かれば、盲目的に逆走することはなくなります。

## ADR とは

Architecture Decision Record は、一つの決定を短い標準フォーマットで書き残す、世界的に定着した習慣です。四つの節でできています。

- **Status** ― その決定が今も有効か(Accepted / Proposed / Superseded など)。
- **Context** ― なぜ決める必要があったのか。当時どんな制約と力が働いていたのか。ここが一番大事な節です。理由はコードには残らないので、未来の読み手が「その前提は今も成り立つか」を判断できるのは、この節があるからです。
- **Decision** ― 何を決めたか。一つの決定を、はっきりと。
- **Consequences** ― その結果、何が楽になり、何が難しくなったか。正直に両面を。受け入れたコストも隠さずに。

`adr-keeper` はこの標準形式で起票します。ただし、すでに ADR を運用しているプロジェクトでは、そのプロジェクトの形式・連番・置き場所に従います。既存の兄弟たちと揃っていることのほうが、このプラグインの流儀に従うことより大事だからです。揃っていない記録は、チームが探す場所に無いのと同じです。

## 使い方

**`/adr-keeper:record <決定の要約>`** で決定を起票します。要約を渡してもいいし、省略すれば直前の会話から決定を拾い、「これで合ってますか」と確認してから書きます。番号は既存の最大+1、置き場所とフォーマットは既存の運用に合わせます。

**`/adr-keeper:list`** で既存の ADR を一覧します。番号・タイトル・状態が並ぶので、「また決める」前に「もう決まっていないか」を確かめられます。ある決定が別の決定を上書きしている(superseded)関係も見えるようにします。

スキルも同梱しています。あなたがコマンドを呼ばなくても、会話の中で方向性の決定が下された瞬間――選択肢の議論が収束し、あなたが「これでいく」と宣言したとき――Claude が「今の、ADR にしておきますか」と控えめに一言添えることがあります。乗らなければ無視して構いません。

## すべての決定を記録するわけではない

これは大事な設計思想です。ADR は「注意」という通貨を消費します。棚に並ぶ記録が増えるほど、未来の読み手が目を通す量も増える。だから `adr-keeper` が拾うのは「未来のセッションが、知らずに逆走したら害になる決定」だけです。

拾うのは、後の作業を縛る方向性の決定――構造の選択、プロジェクトの規約、明示的に受け入れたトレードオフ、そして過去の決定の覆し。拾わないのは、タスクから自然に決まる実装上の選択、可逆な実験、既存のルールがすでに定めていること、コードを読めば分かる事実、チェックで機械的に強制できる制約。後者を ADR にすると、本当に大事な決定が些末な記録の山に埋もれて、誰も棚を見なくなります。記録しないことが正しい場合がある、というのがこのプラグインの節度です。

## できないこと・正直な制約

- **コミットはしません。** ファイルを書くところまでが仕事で、いつコミットするかはあなたの領分です。
- **勝手には起票しません。** スキルは「起票しませんか」と提案するだけで、あなたの承認なしに記録を作りません。
- **決定を代わりに下すことはしません。** ADR は下された決定を書き残す道具であって、何を決めるべきかを決める道具ではありません。理由が会話に無ければ、もっともらしい理由をでっち上げず、分かっていることだけを正直に書きます。
- **1記録につき1決定です。** 会話で二つの独立した決定がついたら、二つの ADR に分けます。混ぜると、後で探すのも上書きするのも難しくなります。
