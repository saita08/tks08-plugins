# md-humanize

Claude は Markdown で書きます。それが一番速くて安いからです。でも生の Markdown は、`#` や `*` や記号が本文に混ざり、長いコードが画面を埋めて、人間にはなかなかの苦行です。AI にとって書きやすい形と、人間にとって読みやすい形は、違うのです。

md-humanize はその間を埋めます。Claude が `.md` を書くたび、同じ場所に読みやすい HTML ページを自動で作ります。「はじめから AI に HTML を書かせればいい」と思うかもしれませんが、それをやると出力トークンが数倍に膨らみます。書くのは安い Markdown のまま、読みやすくする仕事は機械的な変換に任せる——AI は呼ばないのでトークン消費はゼロです。

## 何が起きるか

Claude が `.md` ファイルを作成・編集すると、隣に `<名前>.preview.html` ができます。ブラウザで開くだけです。

- 左に章マップが常駐します。読んでいる場所が点灯し、クリックで移動できます
- 最初の画面は表題と前書きの入口。章が変わる場所には区切りのページが入ります
- それぞれの節はカードになり、広い画面では 2 列に並びます。コードや引用が主役の節は 1 列を使います
- コードブロックには言語名と「コピー」ボタン。1,100 字を超える長いコードや引用は畳んだ状態で置かれ、クリックで開きます
- 上端のバーで読み進み具合がわかります
- ライト / ダーク対応(OS の設定に自動で追従)
- 1 ファイルで完結し、ネット接続なしで開けます。そのまま人に渡せます

生成された HTML がコミットに紛れ込まないよう、最初の生成時にリポジトリの `.gitignore` へ `*.preview.html` を追記します(追記したときはセッションにお知らせが出ます)。

## 見た目とレイアウト

ねらいは「じっくり読ませる」ことではなく、読む人の認知負荷を下げることです。生の Markdown は記号が本文に混ざり、長いコードが画面を埋めて、構造が頭に入るまでが遠い。そこで縦に流れる文書のまま装飾するのではなく、構造を解釈してページのレイアウトへ写します。左に章マップ、最初に表題と前書き、章の変わり目に区切り、節はカードとして並べる、という配置です。

色は白基調のニュートラルに置き、アクセントの青は現在地や進捗バーといった目印にだけ使います。コードだけは常に暗いパネルに載せ、本文と一目で区別がつくようにしています。装飾や演出は持ちません。

## 生成物の扱い

- 元の md が編集されるたびに、同じ preview ファイルへ上書きで再生成されます。生成物は常に元の md の最新の姿です
- 生成物は人間が見るためのもので、Claude のコンテキストを消費させません。Claude が `*.preview.html` を Read / Edit / Write で触ろうとしたときはもちろん、Bash の `cat` や `grep` にファイル名を指定して読もうとしたときもフックが却下し、代わりに元の md を読むよう促します
- ファイル名を指定しない検索については、Grep ツールが `.gitignore` を尊重するため、初回生成時に追記される `*.preview.html` のパターンが生成物を検索対象から外します。Bash での再帰 grep だけは仕組み上塞ぎきれません
- `rm` や `mv`、ブラウザで開く `open` のような、内容を読み込まない操作は妨げません

## 変換しないもの

読み物ではないファイルには手を出しません。

- `CLAUDE.md`、`AGENTS.md`、`README.md`、`CHANGELOG.md`、`LICENSE.md`、`CONTRIBUTING.md`、`CODE_OF_CONDUCT.md`、`SECURITY.md`
- `.claude/`、`.git/`、`node_modules/` の中
- 作業中のプロジェクトの外にあるファイル

## 必要なもの

- Node.js(無い環境では何もせず静かに終わります)

## 中身

```
hooks/hooks.json         Write / Edit 後の生成と、生成物への Read / Edit / Write の却下
scripts/run.sh           node の有無を確認して generate.mjs へ
scripts/generate.mjs     対象の判定 → Markdown を HTML に変換 → 隣に保存
scripts/deny-preview.sh  node の有無を確認して deny-preview.mjs へ
scripts/deny-preview.mjs 生成物への Read / Edit / Write を却下
assets/template.html     ページの見た目とレイアウト構築(CSS / JS 同梱、1 ファイル完結)
vendor/                  変換ライブラリ本体と各ライセンス全文
                         (marked v12: MIT / highlight.js v11: BSD-3-Clause)
```

変換はいつ実行しても同じ結果になる機械的な処理で、失敗してもセッションの進行は妨げません。
