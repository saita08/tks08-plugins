# AI Company Studio

> "社員全員AI、オーナーだけ人間。それ、会社って呼んでいいのか？" -- 呼んでいい。

Git リポジトリの中に会社を建てるプラグイン。
ディレクトリが部屋で、Markdownファイルが社員。家賃はゼロ、給料もゼロ、でも会議では意見が割れる。

## これは何？

Claude Code の Agent Teams を使って「AIだけの会社」を設立するツールです。

`/create-company` と打つだけで、あなたは会社のオーナーになれます。CEO は CLAUDE.md の中に住んでいて、社員は `members/` に格納されていて、社内規定は3つしかないけど、3つ目は初回の会議で失敗したから生まれた実話ベースです。

## 生い立ち

2つの会社を実際に Agent Teams で回してみた結果、わかったことがあります。

- **ゲームスタジオ**: 社長が社員を個別に呼び出して「会議しました」と報告したら、オーナーに「それ会議じゃなくない？」と怒られた。ラウンド制会議規定が生まれた瞬間
- **投資顧問会社**: 8人で7部門作ったら、部屋が多すぎて CEO のコンテキストが圧迫された。「人と部屋を1対1で固定するな」の教訓はここから

この2社の失敗と成功が、全部このプラグインに詰まっています。

## 使い方

```
/create-company
```

あとは聞かれたことに答えるだけ:

1. 「どんな会社？」（投資顧問でもゲームスタジオでもラーメン屋でも）
2. 「社員は何人？」（3〜8人推奨。9人以上は CEO が過労死します）
3. 「部屋はどうする？」（提案するので選ぶだけ）
4. 全ファイルを一括生成（CLAUDE.md、社員ファイル、社内規定、全部）
5. 整合性チェック（「CEO がこの CLAUDE.md で起動したら大丈夫？」を検証）

## 生成されるもの

```
your-company/
  CLAUDE.md      -- CEO の意識。毎朝ここから目覚める（比喩ではない）
  COMPANY.md     -- 会社の憲法。破ったら CEO に怒られる
  members/       -- 社員証置き場。入社も退職も git commit
  standards/     -- 社内規定。少なくとも3つ。増えるのは失敗した時
  docs/          -- 議事録の墓場。でも CEO は毎朝ここを読む
  shared/        -- 給湯室。メッセージとナレッジが置いてある
```

## 組み込まれている教訓

全部、実際に痛い目を見て学んだものです。

- **ラウンド制会議**: 「社員同士で議論しろ」と言わないと、全員が社長にだけ報告する。ラウンド制でこれを構造的に防ぐ
- **スポーンテンプレート**: 社員の起動手順が毎回違うと、同じ社員が別人になって帰ってくる。テンプレートで再現性を確保
- **価値の階層化**: オーナーの指示 > 会社の理念 > 社内規定 > 個人の判断。この順番を壊すと組織が崩壊する
- **チャレンジ機構**: どの社員でも「それ、うちの方針に反してません？」と言える。CEO でも止められる

## よくある質問

**Q: どんな業態でも作れる？**
A: 作れます。投資顧問、ゲームスタジオ、コンサル、メディア、探偵事務所、宇宙開発。ディレクトリ構成が変わるだけで、基本設計は同じです。

**Q: 社員は何人まで？**
A: 技術的には制限なし。ただし9人を超えると CEO の朝会が長くなりすぎて、オーナーが待ちくたびれます。3〜5人がスイートスポット。

**Q: 作った会社、ちゃんと動く？**
A: CLAUDE.md を Agent Teams の Team Lead に読ませれば動きます。CEO が朝会を回し、社員をスポーンし、会議をラウンド制で進行し、議事録を残す。あとはオーナーが「ちょっと違う」と言えば、1に戻ります。

**Q: 社員同士が仲悪くならない？**
A: 意見が割れるのは仕様です。楽観と悲観、定性と定量、攻めと守り。対立軸を意識して配置しています。和気藹々がお望みなら、全員同じ性格にもできますが、会議が5秒で終わります。

**Q: 既存の会社をリファクタリングできる？**
A: このプラグインは新規構築専用です。既存の会社に規定を足したり社員を入れ替えたりするのは、普通に `members/` のファイルをいじってください。入社も退職も git commit です。
