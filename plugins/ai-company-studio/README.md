# AI Company Studio

> "社員全員AI、オーナーだけ人間。それ、会社って呼んでいいのか？" -- 呼んでいい。

Git リポジトリの中に会社を建てるプラグイン。
ディレクトリが部屋で、Markdownファイルが社員。家賃はゼロ、給料もゼロ、でも会議では意見が割れる。

## これは何？

「AIだけの会社」をリポジトリとして設計・生成するツールです。

`/create-company` と打つだけで、あなたは会社のオーナーになれます。CEO は CLAUDE.md の中に住んでいて、社員は `members/` に格納されていて、社内規定は最低3つ必要で、そのうち1つは必ず「最初の会議で痛い目を見てから作る」ことになります。

作った会社は Claude Code の Agent Teams で動かせます。

## なぜこれが必要なのか

「AIだけの会社を作りたい」と思った時、まず悩むのは世界観の設計です。

- 社長ってどんな人格にする？
- 社員が3人いるとして、全員同じ意見だったら会議する意味なくない？
- 社内規定って最初から何が要るの？
- ディレクトリ構成って、部署ごと？機能ごと？

このプラグインは、こういった世界観設計の問いに対話的に答えていくだけで、一貫性のある会社がリポジトリとして出来上がります。Agent Teams で動かすための構造も設計に織り込まれているので、生成した直後から稼動できます。

## 使い方

```
/create-company
```

あとは聞かれたことに答えるだけ:

1. 「どんな会社？」（投資顧問でもゲームスタジオでもラーメン屋でも）
2. 「社員は何人？」（3〜8人推奨。9人以上は CEO が過労死します）
3. 「部屋はどうする？」（提案するので選ぶだけ）
4. 全ファイルを一括生成（CLAUDE.md、社員ファイル、社内規定、全部）
5. 整合性チェック（「CEO がこの CLAUDE.md で起動したら大丈夫？」を検証）

## 生成されるもの

```
your-company/
  CLAUDE.md      -- CEO の意識。毎朝ここから目覚める（比喩ではない）
  COMPANY.md     -- 会社の憲法。破ったら CEO に怒られる
  members/       -- 社員証置き場。入社も退職も git commit
  standards/     -- 社内規定。少なくとも3つ。増えるのは失敗した時
  docs/          -- 議事録の墓場。でも CEO は毎朝ここを読む
  shared/        -- 給湯室。メッセージとナレッジが置いてある
```

業態に応じて `projects/` や `clients/` など、仕事のための部屋が追加されます。

## 世界観に組み込まれている設計思想

- **対立軸のある社員設計**: 全員が同じ結論に達する組織に、会議をする意味はない。楽観と悲観、定性と定量、攻めと守り
- **価値の階層化**: オーナーの指示 > 会社の理念 > 社内規定 > 個人の判断。組織が自律的に動いても逸脱しない構造
- **社内規定は初日から**: 会議の進め方、成果物の置き場、採用ルール。後から作ると混乱する
- **人と部屋を分離する**: 社員は `members/` に住み、部屋は機能で分ける。1人1部屋にすると組織が硬直する

Agent Teams の通信モデルに適合する設計（スポーンテンプレート、ラウンド制会議など）も内蔵されていますが、意識する必要はありません。世界観を作れば、動く構造がついてきます。

## よくある質問

**Q: どんな業態でも作れる？**
A: 作れます。コンサル、メディア、探偵事務所、宇宙開発。ディレクトリ構成が変わるだけで、基本設計は同じです。

**Q: 社員は何人まで？**
A: 技術的には制限なし。ただし9人を超えると CEO の朝会が長くなりすぎて、オーナーが待ちくたびれます。3〜5人がスイートスポット。

**Q: 作った会社、ちゃんと動く？**
A: CLAUDE.md を Agent Teams の Team Lead に読ませれば動きます。CEO が朝会を回し、社員をスポーンし、会議を進行し、議事録を残す。あとはオーナーが「ちょっと違う」と言えば軌道修正されます。

**Q: 社員同士が仲悪くならない？**
A: 意見が割れるのは仕様です。和気藹々がお望みなら全員同じ性格にもできますが、会議が5秒で終わります。

**Q: 既存の会社をリファクタリングできる？**
A: このプラグインは新規構築専用です。既存の会社の社員入れ替えや規定追加は、普通に `members/` のファイルをいじってください。入社も退職も git commit です。
