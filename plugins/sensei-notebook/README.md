# sensei-notebook

> 「ところで、asyncRewake って何?」
> 「かくかくしかじか、こういう仕組みです」
> 「なるほど、わかった」
> （二週間後、別のセッションにて)
> 「ところで、asyncRewake って何?」

概念的な質問への答えは、毎回きれいに使い捨てられます。あなたが「なるほど」と膝を打った説明も、セッションが終われば黒板ごと消され、次に同じ疑問が湧いたとき、Claude はまたゼロから板書を始め、あなたはまた同じ授業を最初から受けることになります。`sensei-notebook` は、腑に落ちた説明を Q&A ノートに書き取らせて、学びを複利で積み上げる仕組みです。良い教師の仕事は、同じ授業を二度しないことです。

## これは何？

「〜って何?」「〜と〜は何が違うの?」「なぜこうなってるの?」。こうした概念的な問いへの答えは、コードにも履歴にも残りません。会話の中で生まれ、会話とともに黒板消しで拭われます。だから同じ質問が、セッションを跨いで何度も繰り返される。優秀な生徒が、毎回記憶を消されて同じ質問をするようなものです。

`sensei-notebook` は、あなたが納得した説明の要点を、プロジェクトのノート(`.claude/sensei-notebook.local.md`)に貯めていきます。そして次に同じ領域の質問が来たとき、Claude はまずそのノートを開いてから答えます。ゼロからの再講義ではなく、前回の理解の上に一段積む。それがこのプラグインの狙う「複利」です。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Skill | `sensei-capture` | 概念質問が来たらノートを参照して答え、納得した Q&A の記録を提案する |
| Command | `/sensei-notebook:quiz` | ノートからランダムに出題する抜き打ちテスト。読むだけで、書き換えはしない |
| Command | `/sensei-notebook:list` | これまで貯めた学びを一覧する |

このプラグインには、セッション開始時に注入する Hook がありません。それは物忘れではなく、教師の方針です。ノートは「質問が来たときに開く」性質のもので、毎朝すべての板書を教室に貼り出すのは、ただ黒板を埋めるだけだからです。だから注入層を持たず、概念質問が来たそのときに、Claude がノートを開きます。

## 仕組み

### 参照 — 質問が来たら、まずノートを開く

あなたが概念的な質問をすると、Claude は答える前にノートを覗きます。関連する項目があれば、その答えを土台にして、今回の問いに合わせて話を広げます。無ければ、普段どおり答えます。この参照は静かに行われます。「ノートを確認しました」とは言いません。授業の準備を生徒の前で読み上げる教師はいません。ただ、答えが少し良くなるだけです。

### 記録 — 腑に落ちたら、ノートに残すか聞く

あなたが「なるほど」「わかった」と納得したとき、Claude は「この Q&A、次のために書き留めておきますか?」と、残そうとしている要点を見せて確認します。頷けば、質問と答えの勘所(会話まるごとではなく、蒸留した要点)がノートに書き取られます。勝手には書きません。あなたが載せると言ったものだけが残ります。板書の丸写しではなく、要点だけを控えるのが、後で読み返せるノートの条件です。

すべての答えを記録するわけではありません。また訊きそうな概念、説明に骨が折れた区別、後の質問の土台になりそうな理解。そういう「後で効く」ものだけを選びます。当たり前のことや一度きりの話は載せません。良いノートは薄いから読み返されるのであって、なんでも書き取れば二度と開かれなくなります。板書を全部写した生徒のノートが、試験前に一度も開かれないのと同じです。

### 復習 — ときどき先生が抜き打ちで

`/sensei-notebook:quiz` を呼ぶと、ノートからランダムに数問、少し芝居がかった老先生が抜き打ちテストを出してきます。数を指定しなければ3問。30秒の遊びですが、貯めた学びを頭の引き出しに戻すのに、なかなか効きます。先生は芝居がかっていても、出題の中身はノートに忠実で、間違ったことは言いません。`/sensei-notebook:list` では、これまで貯めた学びを一覧できます。

## 使い方

インストールすれば、参照層は黙って働き始めます。あなたが概念的な質問をすると、Claude は過去のノートを踏まえて答えるようになります。納得したら、Claude が「残しておきますか?」と聞いてくるので、良ければ頷いてください。

たまに復習したくなったら `/sensei-notebook:quiz`。何を貯めたか見たいときは `/sensei-notebook:list`。ノートそのものは `.claude/sensei-notebook.local.md` にあるので、直接開いて読むのも直すのも自由です。

## ノートはプロジェクトの中に置かれる

ノートは `.claude/sensei-notebook.local.md`、つまりあなたのプロジェクトの中に住みます。Claude だけが読む私的メモではありません。学びは、あなたとチームの見える場所に積まれるべきだからです。チームで使うならバージョン管理に入れれば、プロジェクトの「よくある疑問と答え」が自然に育っていきます。手元だけに留めたいなら `.gitignore` へ。

## よくある質問

**Q: 納得した瞬間に、自動でノートに残りますか?**
A: 「いまユーザーが腑に落ちた」と気づくのは会話の流れを読む Claude の判断です。あなたが明確に反応しなければ、記録の提案は出ないことがあります。生徒がうなずいてくれないと、教師にもそれが伝わったか分からないのと同じです。

**Q: どんな質問でもノートが効きますか?**
A: 参照は「同じ領域の質問」に効きます。ノートに近い項目があれば土台にしますが、まったく新しい話題には土台がありません。ノートは育つほど効いてきます。一学期目の教師より、三学期目の教師のほうが授業が上手いのと同じ理屈です。

**Q: 他の台帳系プラグインみたいに、毎回思い出させてくれないんですか?**
A: それは意図的な設計です。ノートは質問時に開くもので、常時文脈に貼り出す台帳ではありません。「なぜこの構造にしたか」を毎セッション思い出させたいような使い方には、注入型の台帳プラグインのほうが向いています。教科の違いには、違う先生を。

**Q: ノートが分厚くなってきました。**
A: ノートが肥大すると、参照が鈍ります。一項目一概念を守り、当たり前のことを載せないのは、そのためです。ときどき `/sensei-notebook:list` で見直し、要らない項目は手で消してください。ノートの整理も、復習のうちです。

良い教師は、生徒が同じ質問を二度しないように教えます。このノートは、Claude を少しずつそういう教師に近づけるための、たった一冊の連絡帳です。
