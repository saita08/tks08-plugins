# sensei-notebook

> 「ところで、asyncRewake って何?」
> 「かくかくしかじか、こういう仕組みです」
> 「なるほど、わかった」
> （二週間後、別のセッションにて)
> 「ところで、asyncRewake って何?」

概念的な質問への答えは、毎回きれいに使い捨てられます。あなたが「なるほど」と膝を打った説明も、セッションが終われば消え、次に同じ疑問が湧いたとき、Claude はまたゼロから説明を組み立て、あなたはまた同じ話を最初から聞くことになります。`sensei-notebook` は、腑に落ちた説明を Q&A ノートに書き留めて、学びを複利で積み上げていく仕組みです。

## これは何？

「〜って何?」「〜と〜は何が違うの?」「なぜこうなってるの?」——こうした概念的な問いへの答えは、コードにも履歴にも残りません。会話の中で生まれ、会話とともに消えます。だから同じ質問が、セッションを跨いで何度も繰り返されます。

`sensei-notebook` は、あなたが納得した説明の要点を、プロジェクトのノート(`.claude/sensei-notebook.local.md`)に貯めていきます。そして次に同じ領域の質問が来たとき、Claude はまずそのノートを見て、過去の説明を土台に答えます。ゼロからの再説明ではなく、前回の理解の上に一段積む——それがこのプラグインの狙う「複利」です。

## 仕組み

このプラグインは、他の台帳系プラグインと少し違います。**セッション開始時に何かを注入したりはしません。** ノートは「質問が来たときに参照する」性質のもので、毎回全部を文脈に流し込むと重いだけだからです。代わりに、概念質問が来たそのときに、Claude がノートを開きます。

### 参照 — 質問が来たら、まずノートを見る

あなたが概念的な質問をすると、Claude は答える前にノートを覗きます。関連する項目があれば、その答えを土台にして、今回の問いに合わせて話を広げます。無ければ、普段どおり答えます。この参照は静かに行われます。「ノートを確認しました」とは言いません。ただ、答えが少し良くなるだけです。

### 記録 — 腑に落ちたら、ノートに残すか聞く

あなたが「なるほど」「わかった」と納得したとき、Claude は「この Q&A、次のために書き留めておきますか?」と、残そうとしている要点を見せて確認します。頷けば、質問と答えの要点(会話まるごとではなく、蒸留した勘所)がノートに刻まれます。勝手には書きません。あなたが載せると言ったものだけが残ります。

すべての答えを記録するわけではありません。また訊きそうな概念、説明に骨が折れた区別、後の質問の土台になりそうな理解——そういう「後で効く」ものだけを選びます。当たり前のことや一度きりの話は載せません。ノートは絞られているから使い物になるのであって、なんでも貯めれば読まれなくなります。

### 復習 — ときどき先生が抜き打ちで

`/sensei-notebook:quiz` を呼ぶと、ノートからランダムに数問、ちょっと芝居がかった先生が抜き打ちテストを出してきます。30秒の遊びですが、貯めた学びを頭の引き出しに戻すのに、なかなか効きます。`/sensei-notebook:list` では、これまで貯めた学びを一覧できます。

## 使い方

インストールすれば、参照層は黙って働き始めます。あなたが概念的な質問をすると、Claude は過去のノートを踏まえて答えるようになります。納得したら、Claude が「残しておきますか?」と聞いてくるので、良ければ頷いてください。

たまに復習したくなったら `/sensei-notebook:quiz`。何を貯めたか見たいときは `/sensei-notebook:list`。ノートそのものは `.claude/sensei-notebook.local.md` にあるので、直接開いて読むのも直すのも自由です。

## ノートはプロジェクトの中に置かれる

ノートは `.claude/sensei-notebook.local.md`、つまりあなたのプロジェクトの中に住みます。Claude だけが読む私的メモではありません。学びは、あなたとチームの見える場所に積まれるべきだからです。チームで使うならバージョン管理に入れれば、プロジェクトの「よくある疑問と答え」が自然に育っていきます。手元だけに留めたいなら `.gitignore` へ。

## できないこと・正直な制約

- **納得を自動では判定しません。** 「いまユーザーが腑に落ちた」と気づくのは会話の流れを読む Claude の判断です。あなたが明確に反応しなければ、記録の提案は出ないことがあります。
- **参照は「同じ領域の質問」に効きます。** ノートに近い項目があれば土台にしますが、まったく新しい話題には土台がありません。ノートは育つほど効いてきます。
- **毎セッションの自動注入はしません。** これは意図的な設計です。ノートは質問時に参照されるもので、常時文脈に載せる台帳ではありません。「なぜこの構造にしたか」を毎回思い出させたいような使い方には、注入型の台帳プラグインのほうが向いています。
- **ノートが肥大すると、参照が鈍ります。** 一項目一概念を守り、当たり前のことを載せないのは、そのためです。ときどき `/sensei-notebook:list` で見直し、要らない項目は手で消してください。
