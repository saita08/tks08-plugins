# baton

> 「実装できました！」
> 「お、じゃあ実機で確認しとくわ」
> （三日後）
> 「……そういえばあれ、確認したっけ？」

Claude が「終わりました」と言った瞬間、仕事は半分しか終わっていません。残りの半分――実機で動かす、本番に反映する、目視で確かめる――は人間の担当です。そしてこの引き継ぎは、たいてい口頭の一言で済まされ、口頭の一言は落ちます。手順を一つ飛ばし、見るべきでないところを見て、あるいは確認そのものが忘れられて、なんとなく本番に乗る。

`baton` は、その落ちるバトンを、書いて手渡すプラグインです。

## これは何？

Claude の作業が完了しても、Claude 自身には検証できない部分が残ることがあります。デバイスに焼いて動かす。本番のスイッチを入れる。ブラウザで見て「うん、これでいい」と頷く。物理的な結果を確かめる。どれも人間にしかできません。

問題は、「Claude 完了」と「人間が検証」の間にある隙間です。ここは普通、会話の中の一文で埋められます。そして一文は消えます。`baton` は、この隙間に必ず橋を架けます。完了時に「あなたがやること」を、コピペできる手順書として書き出す。前提（何が終わっているか）、手順（番号付き、そのまま実行できる形）、確認観点（何を見れば成功と言えるか）、そして失敗したときの戻し方。これがバトンです。

## 思想: 報告と検証の間には、必ず橋を架ける

「作業完了の報告」と「人間の検証開始」は、別々の出来事です。そしてこの二つの間の空白こそ、仕事が静かに失敗する場所です。何かが間違っていたからではなく、バトンパスそのものが行われなかったから。

だから `baton` の立場は単純です。人間の検証が要る作業は、その検証の仕方を人間が思い出しながら再構築するのではなく、Claude が書いて渡す。完了した作業は、人間が「どう確かめればいいか」を正確に知るまで、まだ完了していません。その知識は、終わらせた Claude の側にあるのだから、Claude が手渡すべきものです。

## 二つの部品

### コマンド — バトンを書く

`/baton:write` を呼ぶと、直前の作業についての手順書が生成されます。書き出し先は、プロジェクトに `notes/` があればそこへ日付つきで、なければ `.claude/baton.local.md` へ。

手順は、人間がそのまま実行できる形で書かれます。「動くか確認して」ではなく、「このコマンドを貼る」「このURLを開く」「この画面のここを見る」。確認観点も、「たぶん大丈夫」ではなく「ログにXが出れば成功」「LEDが緑になれば成功」。そして戻し方。失敗したとき元に戻す正確な手順まで書きます。戻せない変更なら、戻せないと正直に書きます。

### スキル — バトンを促す

人間側の検証が必要な作業を Claude が完了したとき、Claude は「終わりました」で止まりません。同じ息で「実機での確認が要ります。手順書を書いておきますか」と、人間の担当分を名指しして、バトンの作成を促します。あなたが乗ればコマンドが走り、断ればあなたのやり方で渡してください。橋を架けることを申し出るのがスキルの仕事で、渡るかどうかはあなたが決めます。

## なぜフックが無いのか

`baton` にはフックがありません。「作業が完了した」という判定を、機械が確実に下すのは難しいからです。テストが通ったから完了とは限らないし、Claude が長い返答をしたから完了とも限りません。完了かどうか、そして人間の検証が要るかどうかは、文脈を読んでいる Claude 自身が判断するのが最も正確です。だから完了の検知はスキル（Claude の判断）に委ね、機械的なフックは置きません。誤って毎回バトンをせがむプラグインは、すぐに無視されるようになり、本当に要るときのバトンまで一緒に無視されます。

## 使い方

インストールすれば、スキルは黙って待機します。人間の検証が要る作業を終えたとき、Claude がバトンを提案します。自分から書かせたいときは `/baton:write` を直接呼んでください。特に見てほしい観点があれば、引数で渡せます。

## できないこと・正直な制約

- **バトンは検証を代行しません。** これは「あなたがやること」を書き出すツールであって、実機確認や本番反映そのものを肩代わりするものではありません。橋は架けますが、渡るのはあなたです。
- **手順の質は、作業の文脈が Claude に見えている範囲を超えません。** 会話に現れなかった前提や、Claude が触れなかった環境の事情は、バトンにも反映されません。バトンが的外れなら、それは作業がまだ渡せる状態になっていない兆候かもしれません。
- **秘密はバトンに書きません。** バトンはコミットされうるファイルです。トークンや鍵が手順に要るときも、値は書かず、どこで入手するかだけを示します。コミット前に一度目を通してください。
- **バトンは走る記録ではありません。** 毎回まっさらな一枚です。`.claude/baton.local.md` にフォールバックした場合、前のバトンは上書きされます。履歴を残したいなら `notes/` 側に日付つきで貯まるので、そちらを使ってください。
