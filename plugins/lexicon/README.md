# lexicon

> 「"デプロイ"って言ったら本番反映のことね」
> 「はい、ステージングに上げました!」
> 「……いや、それはさっき違うって言ったやつ」
> （翌日、別のセッションにて）
> 「"デプロイ"って言ったら本番反映のことね」

同じ訂正を、人は何度でもする羽目になります。あなたが悪いのではありません。Claude のセッションは毎回まっさらで、昨日交わした「その言葉はこういう意味」という約束を、今日の Claude は知らないからです。`lexicon` は、その約束を書き留めて、次のセッションの Claude にそっと耳打ちしておく係です。

## これは何？

プロジェクトを進めていると、あなたと Claude の間で言葉の意味がズレることがあります。「その"タスク"は Issue のことじゃなくて、TaskCreate のタスクのこと」「"本番"はステージングを含まない」——一度そう訂正すれば、その場は直ります。問題は、その訂正がセッションと一緒に蒸発することです。次の Claude は同じ言葉を、また自己流の意味で使い始めます。

`lexicon` は、確定した語彙をプロジェクトの用語辞書(`.claude/lexicon.local.md`)に貯めていきます。そして新しいセッションが始まるたびに、その辞書を「このプロジェクトの合意済み用語」として Claude に渡します。訂正は一度で済み、以後の全セッションが同じ言葉を話すようになります。

辞書はあなたの見える場所に、人間が読める Markdown の表として置かれます。隠しメモではありません。中身が気に入らなければ、いつでも自分で開いて直せます。

## 仕組み

三つの部品でできています。

### 検知 — 訂正に気づいて、しかし勝手には書かない

会話の中であなたが用語を訂正したり定義したりすると(「〜じゃなくて〜」「この用語の定義が違う」「この意味で統一して」)、Claude はそれを辞書に追記する候補として拾います。ただし、拾ったからといって勝手には書きません。「これを辞書に入れておきますか?」と、書こうとしている一行を見せて確認します。あなたが頷いてから、はじめて一行が刻まれます。

なぜ確認を挟むのか。この辞書は次以降の全セッションに注入されます。間違った一行は、静かに、しかし何度も、未来の Claude に誤りを教え続けます。だから辞書に載るのは「あなたが載せると言ったもの」だけです。

### 注入 — 辞書があるときだけ、静かに渡す

セッションが始まると、`lexicon` はプロジェクトに辞書があるかを見ます。あれば、その中身を「合意済み用語」として Claude の文脈に忍ばせます。無ければ、何もしません。一文字も出力しません。辞書が育ちすぎたときは、直近のものだけを渡すので、文脈が用語表で埋め尽くされることもありません。

### 手動登録 — 会話を待たずに入れたいとき

訂正の場面を待たず、自分から「この用語を登録しておきたい」と思ったら、`/lexicon:add ledger: セッションを跨いで残る追記専用のファイル` のように直接登録できます。この場合は、呼び出したこと自体が承認なので、追加の確認は挟みません。

## 使い方

インストールすれば、注入層は黙って働き始めます。普段どおり作業してください。あなたが用語を訂正すると、Claude が「辞書に入れますか?」と聞いてくるので、良ければ頷いてください。次のセッションから、その言葉は通じます。

自分で入れたいときは `/lexicon:add <用語>: <定義>`。それだけです。

辞書そのものを見たい・直したいときは、`.claude/lexicon.local.md` を開いてください。ただの表なので、行を足すのも消すのも自由です。

## 辞書はプロジェクトの中に置かれる

辞書は `.claude/lexicon.local.md`、つまりあなたのプロジェクトの中に住みます。あなたのマシンの隠しフォルダでも、Claude だけが読む私的メモでもありません。合意した言葉は、あなたとチームの見える場所に置かれるべきだからです。チームで使うなら、この辞書をバージョン管理に入れれば、語彙の合意ごと共有できます。個人的な語彙に留めたいなら、`.gitignore` に入れておけば手元だけに残ります。どちらにするかは、あなたが決めることです。

## 設定

- `LEXICON_MAX_ENTRIES` — 一度のセッションに注入する用語の最大件数(デフォルト 40)。辞書が大きくなったとき、直近この件数だけを渡します。環境変数で調整できます。

## できないこと・正直な制約

- **意味のズレを自動では判定しません。** 「いま用語が訂正された」と気づくのは会話の流れを読む Claude の判断であって、機械的な検知ではありません。あなたが明確に訂正しなかった曖昧なズレは、拾われないことがあります。確実に残したいときは `/lexicon:add` を使ってください。
- **辞書は注入するだけで、強制はしません。** 渡された用語を Claude がどれだけ忠実に守るかは、その場の判断に委ねられます。辞書は「この意味で使ってほしい」という強い示唆であって、機械的な置換ではありません。
- **辞書が肥大すると、古い用語は注入から外れます。** 上限を超えた分は直近優先で切られます。全件を常に効かせたいわけではない、という前提の設計です。どうしても全部を渡したいなら `LEXICON_MAX_ENTRIES` を上げてください。
- **`jq` が必要です。** 注入は `jq` を使って文脈を組み立てます。`jq` が無い環境では、注入は黙ってスキップされます(会話を壊さないための沈黙です)。
