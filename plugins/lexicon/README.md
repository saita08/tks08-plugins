# lexicon

> 「"デプロイ"って言ったら本番反映のことね」
> 「はい、ステージングに上げました!」
> 「……いや、それはさっき違うって言ったやつ」
> （翌日、別のセッションにて）
> 「"デプロイ"って言ったら本番反映のことね」

同じ訂正を、人は何度でもする羽目になります。あなたが悪いのではありません。Claude のセッションは毎回まっさらで、昨日交わした「その言葉はこういう意味」という約束を、今日の Claude は知らないからです。`lexicon` は、その約束を辞書に一行として登録し、次のセッションが始まるたびに黙って机の上に開いておく編纂係です。辞書編纂者に一つだけ矜持があるとすれば、それは「一度合意した語義を、二度と揺らがせない」ことです。

## これは何？

プロジェクトを進めていると、あなたと Claude の間で言葉の意味がズレます。「その"タスク"は Issue じゃなくて TaskCreate のタスク」「"本番"はステージングを含まない」。一度訂正すれば、その場は直ります。問題は、訂正がセッションと一緒に蒸発することです。次の Claude は同じ言葉を、また辞書に載っていない我流の語義で使い始めます。

`lexicon` は、確定した語義をプロジェクトの用語辞書(`.claude/lexicon.local.md`)に見出し語として貯めていきます。そして新しいセッションが始まるたびに、その辞書を「このプロジェクトの合意済み用語」として Claude に渡します。訂正は一度で済み、以後の全セッションが同じ語義で口を揃えます。辞書は同じ見出し語を二度立てません。新しい訂正は既存の行を書き換えるので、矛盾した二つの語義が同じ辞書に並ぶことはありません。改訂版が旧版を上書きする、あの当たり前が働きます。

辞書はあなたの見える場所に、人間が読める Markdown の表として置かれます。隠しメモではありません。編纂に不満があれば、いつでも自分で開いて赤を入れられます。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Skill | `lexicon-capture` | 会話の中の訂正・定義を見出し語の候補として拾い、辞書への登録を提案する |
| Command | `/lexicon:add` | 訂正を待たず、`<用語>: <定義>` の形で語義を直接登録する |
| Hook | SessionStart | 辞書があれば、その中身を合意済み用語としてセッション冒頭に注入する |

## 仕組み

三つの部品でできています。

### 検知 — 訂正に気づいて、しかし勝手には書かない

会話の中であなたが用語を訂正したり定義したりすると(「〜じゃなくて〜」「その用語の定義が違う」「この意味で統一して」)、Claude はそれを見出し語の候補として拾います。ただし、拾った端から辞書に刷るような真似はしません。「これを辞書に入れておきますか?」と、組もうとしている一行を先に見せて確認します。あなたが頷いてから、はじめて活字が組まれます。

なぜ確認を挟むのか。この辞書は次以降の全セッションに注入されます。誤った一行は、静かに、しかし何度も、未来の Claude に間違った語義を教え続けます。校正を経ていない見出し語ほど始末に負えないものはありません。だから辞書に載るのは「あなたが載せると言ったもの」だけです。

### 注入 — 辞書があるときだけ、静かに開く

セッションが始まると、`lexicon` はプロジェクトに辞書があるかを見ます。あれば、その中身を「合意済み用語」として Claude の文脈に忍ばせます。無ければ、何もしません。一文字も出力しません。辞書は分厚くなっても全ページを机に広げたりはせず、直近の見出し語だけを渡すので、文脈が用語表で埋め尽くされることもありません。

### 手動登録 — 訂正の場面を待たずに入れたいとき

訂正を待たず、自分から「この用語を登録しておきたい」と思ったら、`/lexicon:add ledger: セッションを跨いで残る追記専用のファイル` のように直接登録できます。呼び出したこと自体が承認なので、この場合は追加の確認を挟みません。

## 使い方

インストールすれば、注入層は黙って働き始めます。普段どおり作業してください。あなたが用語を訂正すると、Claude が「辞書に入れますか?」と聞いてくるので、良ければ頷いてください。次のセッションから、その言葉は通じます。

自分で入れたいときは `/lexicon:add <用語>: <定義>`。それだけです。

辞書そのものを見たい・直したいときは、`.claude/lexicon.local.md` を開いてください。ただの表なので、行を足すのも消すのも自由です。

## 辞書はプロジェクトの中に置かれる

辞書は `.claude/lexicon.local.md`、つまりあなたのプロジェクトの中に住みます。あなたのマシンの隠しフォルダでも、Claude だけが読む私的メモでもありません。合意した言葉は、あなたとチームの見える場所に置かれるべきだからです。チームで使うなら、この辞書をバージョン管理に入れれば、語彙の合意ごと共有できます。個人的な語彙に留めたいなら、`.gitignore` に入れておけば手元だけに残ります。どちらにするかは、あなたが決めることです。

## 設定

- `LEXICON_MAX_ENTRIES` — 一度のセッションに注入する用語の最大件数(デフォルト 40)。辞書が大きくなったとき、直近この件数だけを渡します。環境変数で調整できます。

## よくある質問

**Q: 辞書に載せたのに、Claude が別の意味で使ってきます。**
A: 辞書は「この意味で使ってほしい」という強い示唆であって、機械的な語句置換ではありません。渡された語義をどれだけ忠実に守るかは、その場の Claude の判断に委ねられます。編纂者は語義を定めますが、話し手の口までは縛れません。それでも、辞書が無い日々よりは遥かにマシです。

**Q: ズレた瞬間に、自動で拾ってくれますか?**
A: 拾うのは「いま用語が訂正された」と会話の流れを読む Claude の判断であって、機械的な検知ではありません。あなたが明確に訂正しなかった曖昧なズレは、見落とされることがあります。確実に一行残したいときは `/lexicon:add` を使ってください。編纂は、原稿が持ち込まれてこそ進みます。

**Q: 古い用語が通じなくなりました。**
A: 辞書が上限を超えると、古い見出し語は直近優先で注入から外れます。全件を常に効かせたいわけではない、という前提の設計です。どうしても全ページを机に広げたいなら `LEXICON_MAX_ENTRIES` を上げてください。

**Q: `jq` が要ると聞きました。**
A: 注入は `jq` を使って文脈を組み立てます。`jq` が無い環境では、注入は黙ってスキップされます。会話を壊してまで語義を主張するほど、この編纂者は無粋ではありません。

辞書とは、揺れる言葉に錨を下ろす道具です。錨を一度下ろしたら、潮が変わっても同じ位置に戻れる。それだけのために、この編纂係は毎朝あなたの机に一冊を開いておきます。
