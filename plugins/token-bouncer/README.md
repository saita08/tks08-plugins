# token-bouncer

> 「エージェントをもう一体、いや念のためもう一体」
> 「さっきのファイル、もう一回読んでおくか」
> 「……ところで今、何万トークン焼けた?」
> 「分からん。気づいたときには、もう焼けている」

トークンの浪費は、事故のように起きるのではなく、習慣のようにじわじわ起きます。エージェントの多重起動、同じファイルの反復読み込み。一つ一つは無害な顔をしていて、合計だけが大きい。そして気づくのはたいてい、勘定書が来てからです。

`token-bouncer` は、その浪費の兆候をリアルタイムで見張る用心棒です。しきい値を超えたら、あなたが何か言う前に、Claude の肩を叩いて「本当に必要か?」と一言そえます。止めはしません。用心棒は追い出す係ではなく、目を光らせる係だからです。

## これは何？

二つのフックが、セッション中の「浪費と相関する振る舞い」を静かに数えます。

**エージェントの多重起動。** `Agent` / `Task` の起動を数え、しきい値(既定 5)に達したら、そしてそれを超えるたびに、Claude に控えめな警告を差し込みます。サブエージェントは一体ごとに独立したセッションで、独立してトークンを焼きます。反射的にもう一体起こす前に、一拍おいてもらうための一言です。

**同じファイルの反復読み込み。** `Read` を見張り、同じファイルが 3 回目(既定)に読まれたら警告します。すでに内容を持っているのに読み直すのは、そのぶんトークンを二重に払う行為です。手元にあるなら、それを使えばいい。

どちらの警告も、ブロックではありません。`additionalContext` として Claude に見える助言を添えるだけで、ツールの実行はそのまま通します。用心棒は判断を下しません。目を光らせて、控えめに声をかけるだけです。

## 勘定書

`/token-bouncer:tab`(勘定書)を呼ぶと、今セッションの集計が出ます。起動したエージェントの数と、複数回読まれたファイルの一覧。溜まっていなければ、勘定書はきれいなままです。

## 設定

しきい値はプロジェクトごとに変えられます。`.claude/token-bouncer.local.md` を置き、YAML frontmatter に書くだけです。

```markdown
---
agent_threshold: 3
read_threshold: 4
---
```

ファイルが無ければ、既定値(エージェント 5、読み込み 3)で動きます。設定が読めなくても、既定値に静かに戻るだけで、会話は汚しません。

## 正直な制約

**これはトークン数そのものを測っていません。** これは、このプラグインが最初に白状しておくべき一番大事なことです。Claude Code のハーネスは、フックにトークン数を渡しません。だから `token-bouncer` に「今いくら焼けたか」は原理的に分かりません。

測っているのは、**浪費と相関する振る舞い**です。エージェントの多重起動も、同じファイルの反復読み込みも、無駄なトークン消費とよく一緒に現れます。だから兆候として見張る価値がある。しかしそれは相関であって、証明ではありません。数が多いことは「一拍おいて考える」きっかけであって、「無駄だった」の証拠ではありません。多重起動が正解のときも、読み直しが必要なときも、当然あります。

- **止めません。** 警告は助言であって、拒否ではありません。ツールは必ずそのまま実行されます。判断はあなたと Claude のものです。
- **セッションをまたぎません。** カウントはセッション単位です。別セッションの起動や読み込みは数えません。
- **勘定書は最新セッションを推測します。** スラッシュコマンドはセッション ID を受け取らないので、`tab` は最後に更新された記録を今のセッションとみなします。ずれていたら、引数でセッション ID を渡せます。

## 構成

```
token-bouncer/
├── .claude-plugin/
│   └── plugin.json
├── hooks/
│   ├── hooks.json          # PreToolUse: Agent|Task と Read の二枚
│   └── count.sh            # カウンタ追記 + しきい値判定。超過時のみ助言を注入
├── commands/
│   ├── tab.md              # 勘定書を提示
│   └── tab.sh              # 今セッションの集計を出す
└── README.md
```

## 前提

- `jq` がパスにあること。フックも集計も jq で JSON を扱います。
- しきい値を変えたい場合のみ `.claude/token-bouncer.local.md`。無くても既定値で動きます。
