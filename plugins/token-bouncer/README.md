# token-bouncer

> 「エージェントをもう一体、いや念のためもう一体」
> 「さっきのファイル、もう一回読んでおくか」
> 「……ところで今、何万トークン焼けた?」
> 「分からん。気づいたときには、もう焼けている」

トークンの浪費は、事故のように起きるのではなく、習慣のようにじわじわ起きます。エージェントの多重起動、同じファイルの反復読み込み。一つ一つは無害な顔で店に入ってきて、合計だけが大きい。そして気づくのはたいてい、勘定書が来てからです。

`token-bouncer` は、その浪費の兆候を夜の店先で見張る用心棒です。しきい値を超えた客が入ろうとすると、あなたが何か言う前に Claude の肩を叩き、「本当に必要か?」と一言そえます。止めはしません。用心棒は追い出す係ではなく、目を光らせる係だからです。

## これは何？

二つのフックが、セッション中の「浪費と相関する振る舞い」を静かに数えます。腕組みして立っているだけに見えて、店に入る客は全員数えています。

**エージェントの多重起動。** `Agent` / `Task` の起動を数え、しきい値(既定 5)に達したら、そしてそれを超えるたびに、Claude に控えめな警告を差し込みます。サブエージェントは一体ごとに独立したセッションで、独立してトークンを焼きます。反射的にもう一体呼ぶ前に、一拍おいてもらうための一声です。

**同じファイルの反復読み込み。** `Read` を見張り、同じファイルが 3 回目(既定)に読まれたら声をかけます。すでに内容を手に持っているのに読み直すのは、同じ入場料を二度払う客と同じです。手元にあるなら、それを使えばいい。

どちらの警告も、ブロックではありません。`additionalContext` として Claude に見える助言を添えるだけで、ツールの実行はそのまま通します。用心棒は判断を下しません。腕は組んでも、腕ずくでは止めない。目を光らせて、控えめに声をかけるだけです。

## コンポーネント

| 種別 | 名前 | 役目 |
|------|------|------|
| Hook | `PreToolUse` (Agent\|Task) | エージェント起動を数え、しきい値超過ごとに一声かける |
| Hook | `PreToolUse` (Read) | 同じファイルの反復読み込みを数え、しきい値で一声かける |
| Command | `/token-bouncer:tab` | 今セッションの勘定書を提示する |

## いちばん先に白状すること

**この用心棒は、トークン数そのものを測っていません。** ここを隠したら詐欺なので、いちばん先に言います。Claude Code のハーネスは、フックにトークン数を渡しません。だから `token-bouncer` に「今いくら焼けたか」は原理的に分かりません。店先に立ってはいるが、レジの中身は見えていない用心棒です。

では何を見張っているのか。**浪費と相関する振る舞い**です。エージェントの多重起動も、同じファイルの反復読み込みも、無駄なトークン消費とよく連れ立って店に入ってきます。だから兆候として見張る価値がある。しかしそれは相関であって、証明ではありません。数が多いのは「一拍おいて考える」きっかけであって、「無駄だった」の証拠ではありません。多重起動が正解のときも、読み直しが必要なときも、当然あります。用心棒は人相で声をかけているのであって、財布の中身を数えているわけではないのです。

- **止めません。** 警告は助言であって、拒否ではありません。ツールは必ずそのまま実行されます。判断はあなたと Claude のものです。
- **セッションをまたぎません。** カウントはセッション単位です。昨夜の客のことは、今夜の用心棒は覚えていません。
- **勘定書は最新セッションを推測します。** スラッシュコマンドはセッション ID を受け取らないので、`tab` は最後に更新された記録を今のセッションとみなします。ずれていたら、引数でセッション ID を渡せます。

## 勘定書

`/token-bouncer:tab`(勘定書)を呼ぶと、今セッションの集計が出ます。起動したエージェントの数と、複数回読まれたファイルの一覧。何も溜まっていなければ、勘定書はきれいなままです。用心棒が暇な夜は、良い夜です。

なお、この勘定書に金額は載りません。載せられません。上に白状したとおり、トークン数は測れないからです。ここに並ぶのは焼けたトークンの合計ではなく、焼けていそうな振る舞いの回数です。金額の代わりに「何回それをやったか」が書いてある伝票だと思ってください。

## 設定

しきい値はプロジェクトごとに変えられます。用心棒の目つきの厳しさを、店ごとに調整できると思ってください。`.claude/token-bouncer.local.md` を置き、YAML frontmatter に書くだけです。

```markdown
---
agent_threshold: 3
read_threshold: 4
---
```

ファイルが無ければ、既定値(エージェント 5、読み込み 3)で動きます。設定が読めなくても、既定値に静かに戻るだけで、会話は汚しません。目つきの設定が壊れていても、用心棒は黙って標準の目つきに戻ります。

## よくある質問

**Q: 結局いくらトークンを使ったのか、これで分かる？**
A: 分かりません。この用心棒はレジの中を見られないと、上で正直に白状したとおりです。分かるのは「エージェントを何体呼んだか」「どのファイルを何回読み直したか」までです。金額ではなく、素振りを見ています。

**Q: 警告が出た。もうエージェントを呼んじゃいけない？**
A: 呼んでいいです。用心棒は止めません。声をかけるだけで、扉は開けたままにしてあります。多重起動が正解の場面はいくらでもあります。ただ、反射で呼ぶ前に一拍だけ置いてくれれば、それでこの店は役目を果たしています。

**Q: しきい値まで使わなければ、無駄はゼロだと保証してくれる？**
A: しません。数が少ないことは無駄がないことの証明ではないし、数が多いことも無駄の証明ではありません。これは相関を見張る仕事で、証明する仕事ではありません。保証する用心棒がいたら、そいつは嘘をついています。

## 構成

```
token-bouncer/
├── .claude-plugin/
│   └── plugin.json
├── hooks/
│   ├── hooks.json          # PreToolUse: Agent|Task と Read の二枚
│   └── count.sh            # カウンタ追記 + しきい値判定。超過時のみ助言を注入
├── commands/
│   ├── tab.md              # 勘定書を提示
│   └── tab.sh              # 今セッションの集計を出す
└── README.md
```

## 前提

- `jq` がパスにあること。フックも集計も jq で JSON を扱います。
- しきい値を変えたい場合のみ `.claude/token-bouncer.local.md`。無くても既定値で動きます。

トークンは数えられない。数えられるのは、トークンを焼きに来た客の顔つきだけだ。
