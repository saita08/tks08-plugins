# claude-rpg

> 深夜2時、6時間ぶっ通しのセッション、ツール呼び出し843回。
> 誰も、褒めてくれない。
> ログには残っている。偉業として、ではなく、ただの数字として。

開発の日常は、数字の上ではれっきとした冒険です。夜を徹して城を築き、何百という呪文（ツール呼び出し）を唱え、季節をまたいで一つのプロジェクトと戦い続ける。RPGなら間違いなく実績が解除される場面が、現実では静かに `stats-cache.json` に沈んでいくだけです。

`claude-rpg` は、その沈んだ数字を掘り起こして、冒険の記録に変えます。ただし、盛りません。あなたが本当にやったことだけを、実績として称えます。

## これは何？

あなたの Claude Code には、これまでの利用がまるごと記録されています。何セッション回したか、何メッセージ交わしたか、一番長いセッションは何時間だったか、何時に働いていたか。`claude-rpg` はその実データを読み、あらかじめ定義された20ほどの実績を一つずつ判定して、「冒険者ステータス」として見せます。

肝は、**すべての実績が実データだけで判定できる**という点です。「夜ふかしの徒」は `hourCounts` の深夜帯にカウントがあるとき。「ウルトラマラソン」は最長セッションが6時間を超えたとき。「七日連続」は稼働日が7日つながったとき。占いでも気分でもなく、記録が言っていることだけを実績にします。だから解除された実績には、必ずそれを裏づけた実際の数字が添えられます。「長時間セッションを達成」ではなく「最長セッション 6h 12m」と。

そして記録が無ければ、無いと正直に言います。まだ何も冒険していない人に、架空の勲章は渡しません。

## 使い方

`/claude-rpg:status` を叩くと、冒険者ステータスが開きます。

- **称号** — 解除した実績の数から決まるランク（見習いから伝説まで）。何個中何個か、必ず添えます。
- **冒険者の記録** — 総セッション数、総メッセージ数、最長セッション、初セッションの日付。全部あなたの実データです。
- **解除済みの実績** — 解除した実績と、それを解除した実際の数字。
- **次に近い実績** — もう少しで届く実績と、あと何セッション・あと何日で届くのか。

解除状態は `~/.claude/claude-rpg/achievements.json` に保存されます。一度解除した実績は取り消されません。あとから古い記録が整理されて数字が減っても、達成した事実は消えない、という約束です。

## セッション開始時のひとこと

SessionStart フックが、セッションが始まるたびに裏で軽く実績を再判定します。**前回から新しく解除された実績が1つでもあれば**、一言だけ祝います。無ければ、完全に黙ります。喋るのは新記録が出た時だけ。それ以外は、いないのと同じです。

このフックは重い判定をしません。「新しく解除されたものがあるか」だけを素早く見て、あれば控えめに祝い、詳しいステータスは `/claude-rpg:status` に譲ります。初回インストール直後は、比較する過去が無いので祝いません（全部が「新規」に見えてうるさいだけなので）。次のセッションから、本当の新記録だけを拾います。

## statusline に実績を出したい人へ

このプラグインは、あなたの statusline を勝手に書き換えません。statusline はユーザーの聖域で、ここに一行足すかどうかはあなたが決めることだからです。

もし自分の statusline スクリプトに「解除数」を出したいなら、次のようなスニペットを自分のスクリプトに組み込んでください。保存済みの `achievements.json` を読むだけなので軽量です。

```sh
# 自分の statusline スクリプトの中で
rpg_file="$HOME/.claude/claude-rpg/achievements.json"
if [ -f "$rpg_file" ] && command -v python3 >/dev/null 2>&1; then
  rpg_count="$(python3 -c 'import json,sys;
try:
    d=json.load(open(sys.argv[1])); print(len(d.get("unlocked",{})))
except Exception: print("")' "$rpg_file" 2>/dev/null)"
  [ -n "$rpg_count" ] && printf 'RPG %s/20  ' "$rpg_count"
fi
```

`/claude-rpg:status` を一度も実行していなくても、SessionStart フックが `achievements.json` を書くので、値は自然に埋まっていきます。

## できないこと・正直な制約

- **記録が無ければ何も言えません。** `~/.claude/stats-cache.json` が存在しない環境（新規インストール直後や、統計を無効化している場合）では、実績は判定できません。そのときは架空の値をでっち上げず、「記録が見つからない」と正直に返します。
- **統計の集計には時差があります。** `stats-cache.json` はある時点までの集計です。たった今のセッションが即座に反映されるとは限りません。実績はその時点で読めた記録に基づきます。
- **判定できる実績は、データにある事実に限られます。** 「難しいバグを倒した」ような質的な偉業は、数字に残らないので実績にできません。ここが称えられるのは、あくまで記録に残る量的な冒険です。
- **`history.jsonl` が大きいと、集計に一瞬かかることがあります。** コマンドの種類数やプロジェクト数はこのファイルから数えるので、巨大な履歴では読み取りに少し時間がかかります（それでも全文をメモリに載せる作りにはしていません）。
- **依存は python3 のみ。** フックと集計は python3 を使います。無い環境ではフックは黙ってスキップします（会話をエラーで汚しません）。

## 構成

```
claude-rpg/
├── .claude-plugin/
│   └── plugin.json
├── commands/
│   └── status.md            # /claude-rpg:status — ステータス表示の入口
├── hooks/
│   ├── hooks.json           # SessionStart フック
│   └── session-greet.sh     # 新規解除があるときだけ一言祝う。無ければ沈黙
├── skills/
│   └── adventurer-status/
│       ├── SKILL.md          # データ読取 → 判定 → 永続化 → ステータス表示
│       └── references/
│           ├── achievements.md     # 約20個の実績と、その判定条件・読むフィールド
│           ├── status-display.md   # ランクとステータスカードの見せ方
│           └── persistence.md      # achievements.json の読取・統合・書込
└── README.md
```

## 前提

- `~/.claude/stats-cache.json` が存在すること（Claude Code の利用統計）。無ければ正直にそう言うだけで、壊れはしません。
- `python3` がパスにあること。フックとデータ集計に使います。
