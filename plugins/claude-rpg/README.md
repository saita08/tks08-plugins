# claude-rpg

> 深夜2時、6時間ぶっ通しのセッション、ツール呼び出し843回。
> 誰も、褒めてくれない。
> ログには残っている。偉業として、ではなく、ただの数字として。

冒険者ギルドへようこそ。ここはあなたの戦歴を査定する窓口です。夜を徹して城を築き、何百という呪文を唱え、季節をまたいで一つのプロジェクトと戦い続けた。RPGなら間違いなく実績が解除される場面が、現実では静かに `stats-cache.json` に沈んでいくだけでした。ギルドはその沈んだ数字を掘り起こし、正式な冒険の記録として台帳に記します。ただし、盛りません。あなたが本当にやったことだけを勲章にします。査定官は無から勲章を作りません。それは詐称であり、ギルドは詐称を最も嫌います。

## これは何？

あなたの Claude Code には、これまでの利用がまるごと記録されています。何セッション回したか、何メッセージ交わしたか、一番長いセッションは何時間だったか、何時に働いていたか。`claude-rpg` はその実データを読み、あらかじめ定義された約20の実績を一つずつ判定して、「冒険者ステータス」として見せます。

肝は、**すべての実績が実データだけで判定できる**という点です。「夜ふかしの徒」は深夜0時から4時のどこかに稼働の記録があるとき解除されます。夜ふかしを自己申告する必要はありません。台帳が知っています。「ウルトラマラソン」は最長セッションが6時間を超えたとき。「不夜城」は24時間のうち20時間帯以上に稼働の痕跡があるとき、つまりほぼ寝ていないときに贈られます。名誉な称号かどうかは、査定官は判断しません。事実だけを記します。

だから解除された実績には、必ずそれを裏づけた実際の数字が添えられます。「長時間セッションを達成」ではなく「最長セッション 6h 12m」と書きます。そして記録が無ければ、無いと正直に言います。まだ何も冒険していない新人に、架空の勲章は渡しません。手ぶらで来た者に「魔王を倒した」と書いた台帳を渡す査定官は、そもそも査定官ではありません。

## 実績台帳の一部

判定される約20の実績は、どれも実データだけで白黒がつくものだけです。気分や自己申告は一つも入りません。抜粋すると、こういう戦歴が刻まれます。

- **「万の言葉」** — 累計メッセージが40000を超えたとき。喋った量は、多いか少ないかではなく、ただ40000を超えたか超えていないかで判定されます。
- **「七日連続」** — 稼働した暦日が7日つながったとき。飛び石は連続とみなしません。休んだ日は、正直に休んだ日として数えます。
- **「多元世界の渡り」** — 履歴に10個以上の異なるプロジェクトが現れたとき。あちこちに手を出したことの、動かぬ証拠です。
- **「コマンド探検家」** — 5種類以上のスラッシュコマンドを実際に使ったとき。ここでいうコマンドは厳密です。`/Users/...` のようなファイルパスは、先頭がスラッシュでもコマンドには数えません。パスを一つ叩いただけで探検家を名乗れては、探検の名がすたるからです。

称号は、いくつ解除したかで決まるランクです。見習いから伝説まで、何個中何個か必ず添えます。「伝説」を名乗るには、伝説ぶんの実績が要ります。自称は通りません。

## 使い方

`/claude-rpg:status` を叩くと、冒険者ステータスが開きます。

- **称号** — 解除した実績の数から決まるランク。何個中何個かを添えます。
- **冒険者の記録** — 総セッション数、総メッセージ数、最長セッション、初セッションの日付。全部あなたの実データです。
- **解除済みの実績** — 解除した実績と、それを解除した実際の数字。
- **次に近い実績** — もう少しで届く実績と、あと何セッション・あと何日で届くのか。「あと3セッションで常連」と、届く距離まで数字で言います。

解除状態は `~/.claude/claude-rpg/achievements.json` に保存されます。一度解除した実績は取り消されません。あとから古い記録が整理されて数字が減っても、達成した事実は消えない、という約束です。ギルドの台帳は、一度書いた戦歴を後から削りません。

## セッション開始時のひとこと

SessionStart フックが、セッションが始まるたびに裏で軽く実績を再判定します。**前回から新しく解除された実績が1つでもあれば**、一言だけ祝います。無ければ、完全に黙ります。喋るのは新記録が出た時だけで、それ以外はいないのと同じです。祝われないセッションは、あなたが何もしていないのではなく、まだ次のしきい値を超えていないだけです。

このフックは重い判定をしません。「新しく解除されたものがあるか」だけを素早く見て、あれば控えめに祝い、詳しいステータスは `/claude-rpg:status` に譲ります。初回インストール直後は、比較する過去が無いので祝いません。過去が無いと全実績が「新規」に見えて、22連発で祝ってくることになり、それは祝福ではなく通知の暴力だからです。次のセッションから、本当の新記録だけを拾います。

## statusline に実績を出したい人へ

このプラグインは、あなたの statusline を勝手に書き換えません。statusline はユーザーの聖域で、ここに一行足すかどうかはあなたが決めることだからです。ギルドは台帳を管理しますが、あなたの家の壁に無断で額縁を掛けたりはしません。

もし自分の statusline スクリプトに「解除数」を出したいなら、次のスニペットを自分のスクリプトに組み込んでください。保存済みの `achievements.json` を読むだけなので軽量です。

```sh
# 自分の statusline スクリプトの中で
rpg_file="$HOME/.claude/claude-rpg/achievements.json"
if [ -f "$rpg_file" ] && command -v python3 >/dev/null 2>&1; then
  rpg_count="$(python3 -c 'import json,sys;
try:
    d=json.load(open(sys.argv[1])); print(len(d.get("unlocked",{})))
except Exception: print("")' "$rpg_file" 2>/dev/null)"
  [ -n "$rpg_count" ] && printf 'RPG %s/20  ' "$rpg_count"
fi
```

`/claude-rpg:status` を一度も実行していなくても、SessionStart フックが `achievements.json` を書くので、値は自然に埋まっていきます。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/claude-rpg:status` | 実データから実績を判定し、冒険者ステータスカードを表示する入口 |
| Skill | `adventurer-status` | データ読取 → 実績判定 → 永続化 → ステータス表示の本体。読まなかったデータは決して主張しない |
| Hook | `SessionStart` | 新しく解除された実績があるときだけ一言祝う。無ければ沈黙 |

## よくある質問

**Q: 数字を盛って、豪華なステータスにしてくれませんか？**
A: できません。それが唯一やらないと決めたことです。査定官は、あなたの記録に無い偉業を台帳に書きません。豪華さが欲しいなら、豪華になるまで働いてください。台帳はその日を待っています。

**Q: 「難しいバグを倒した」を実績にできませんか？ 一番の偉業なのに。**
A: できません。それが数字に残らないからです。質的な偉業は、あなたの記憶には刻まれても `stats-cache.json` には刻まれません。ここが称えられるのは、記録に残る量的な冒険だけです。倒したバグの供養は、別の場所でお願いします。

**Q: 昨日の激闘が反映されていません。**
A: `stats-cache.json` はある時点までの集計なので、たった今のセッションが即座に映るとは限りません。実績はその時点で読めた記録に基づきます。反映は待てば来ます。台帳は逃げません。

**Q: `stats-cache.json` が無い環境だとどうなりますか？**
A: 正直に「記録が見つからない」と返します。架空の値ででっち上げたりはしません。統計を無効化している人や、入れたばかりの人には、まだ台帳に書くことがない、ただそれだけです。

**Q: python3 が無い環境では？**
A: フックは黙ってスキップします。会話をエラーで汚すより、静かに何もしないほうを選びます。冒険は続けられますが、開始時のひとことは鳴りません。

## 前提

- `~/.claude/stats-cache.json` が存在すること。無ければ正直にそう言うだけで、壊れはしません。
- `python3` がパスにあること。フックとデータ集計に使います。

査定官より一言。あなたの戦歴は、あなたが思うより長い。ただ、誰も台帳を開いていなかっただけです。
