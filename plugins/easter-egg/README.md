# easter-egg

> "すべての真面目なプロダクトには、ふざけた秘密が1つ必要だ" -- 誰も言っていない

あなたのコードベースにイースターエッグを仕込むプラグイン。
なぜなら、`git blame` で見つかる最高のものは、バグではなくジョークだから。

## これは何？

真面目に作ったプロダクトに、真面目にイースターエッグを仕込むためのツールです。
矛盾しているようで、していません。Excel 97 にフライトシミュレーターを隠した先人たちも、きっと真面目でした。たぶん。

## 特徴

- **フェロー級の知識**: Atari Adventure (1979) から Chrome の T-Rex まで、イースターエッグの歴史を網羅。HTTP 418 が何の略か即答できるようになります
- **13の実装パターン**: コナミコマンド、ASCII アート、隠し API、日付トリガー... あなたのプロジェクトに最適な仕込み方を提案
- **対話的ワークフロー**: コードベースを分析して「このプロジェクトならこれが面白い」と提案。選ぶだけ
- **言語不問**: Web でも CLI でも API でもモバイルでも。イースターエッグに国境はない

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/easter-egg` | プロジェクトを分析して、イースターエッグを対話的に仕込む |
| Skill | `easter-egg-knowledge` | 実装パターン、名作事例、心理学、ベストプラクティスの知識ベース |

## 使い方

```
/easter-egg
```

あとは AI が勝手にやってくれます:

1. プロジェクトの技術スタックを分析（「React か... なるほど」）
2. そのプロジェクトに合うイースターエッグを3つ以上提案（「Konami コードでコンフェッティとかどう？」）
3. 好みを選択（「2番で」「了解」）
4. ベストプラクティスに従って実装（パフォーマンス、セキュリティ、アクセシビリティ全部守ります。ふざけてるけど真面目なので）

## 知識ベースの中身

`easter-egg-knowledge` スキルは、こういう時に自動で発動します:

- 「イースターエッグを入れたい」
- 「隠し機能を作りたい」
- 「面白い 404 ページにしたい」
- 「コードにユーモアを足したい」

中身は:

- **13種の実装パターン** -- コード例付き。コピペで使える
- **殿堂入り事例集** -- "The Book of Mozilla" を知ってるとモテます（エンジニアに）
- **三原則** -- Zero-Cost Law（発動しなければコスト0）、Delight Law（喜びが目的）、Discovery Law（見つけた時が最高の瞬間）
- **やっちゃいけないこと** -- 「隠し管理画面」はイースターエッグではなくバックドアです

## よくある質問

**Q: プロダクションに入れて大丈夫？**
A: Zero-Cost Law を守れば大丈夫です。発動しない限り存在しないのと同じ。Chrome の T-Rex ゲームだってプロダクションにいます。

**Q: 上司に怒られない？**
A: 責任は持てません。ただし、Excel 97 にフライトシミュレーターを入れた人は解雇されていません。

**Q: イースターエッグのイースターエッグはありますか？**
A: 探してみてください。
