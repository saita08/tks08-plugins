# idea-forge

> あなたが去年からこっち、同じ形のプロンプトを何百回打ってきたか、あなたは知りません。
> どの手順を、毎回手で回しているか。何曜日の何時に、いつも同じ作業に戻ってくるか。
> その記録は、`~/.claude/` の奥に、誰にも読み返されないまま積もっています。

自分の Claude 利用履歴は、次に作るべき自動化の、いちばん正直な設計図です。あなたが本当に何を繰り返しているかは、頭の中の印象ではなく、そこに書いてあります。ただ、誰も掘り返さない。`idea-forge` は、その井戸を掘り直す鍛冶場です。

## これは何？

`pluginize` という姉妹プラグインは、**いま目の前で起きている作業**から「これプラグインにできますよ」と拾います。区切りごとに肩を叩く、現場の観測者です。

`idea-forge` が掘るのは、そこではありません。**蓄積された履歴の井戸**です。今日の作業ではなく、この数ヶ月あなたが積み上げてきた利用の地層。そこには、一日の作業では見えない繰り返しが埋まっています。毎週やっている手作業、いつの間にか定着したコマンドの組み合わせ、特定の時間帯にいつも戻ってくる作業。一回では気づかないものが、百回分の履歴を重ねると、地層の縞模様になって浮かびます。

鍛冶場は、その縞模様を読み、プラグイン候補を**根拠つきで**鍛え上げて差し出します。「なんとなく便利そう」ではなく、「あなたはこの形のプロンプトを41回打っていて、うち大半は同じ三手順で、既存のどのプラグインもこれを回していない」という、履歴に裏打ちされた提案です。

## 何を掘るのか

鍛冶場が掘り返す地層は、四つあります。

- **コマンドの頻度とプロンプトの型** — `~/.claude/history.jsonl` に残るあなたの入力の全ログ。どのスラッシュコマンドを何回、どんな形のプロンプトを繰り返し打っているか。同じ骨格のプロンプトが何十回も現れれば、それは自動化されたがっている作業です。
- **利用のリズム** — `~/.claude/stats-cache.json` に集計された、日ごと・時間帯ごとの活動。いつ、どれくらい、どのモデルで働いているか。リズムには、定期作業の影が映ります。
- **繰り返しの手作業** — 直近トランスクリプトのサンプリングから、手で回している段取りや、毎回同じ順で踏んでいる手順を拾います。
- **すでに持っているもの** — 導入済みのプラグインと組み込み機能。候補がこれらと重複していないかを突き合わせ、「もうあるもの」を提案から外します。井戸を掘って、すでに汲んである水をもう一度差し出すのは、鍛冶場の恥です。

## 使い方

`/idea-forge:dig` を実行してください。鍛冶場は前回の続きから履歴を掘り、四つの地層を読み、候補を絞り込み、**根拠の濃い順に**並べて提案します。

各候補には、なぜそれが候補なのかの根拠がつきます。どの地層の、どんな観測から来たのか。頻度なのか、リズムなのか、手作業の繰り返しなのか。根拠が濃いものほど上に来ます。薄い思いつきは、下に沈むか、そもそも出てきません。

## 鍛冶場は提案する。作るのは別の職人

鍛冶場の仕事は、候補を鍛えて差し出すところまでです。**プラグインそのものは作りません。**

作る段になったら、専門の職人に渡してください。目の前の作業から拾う `pluginize`、あるいは対話でプラグインを組み上げる `plugin-dev`。鍛冶場は「何を作るべきか」を根拠つきで示す係で、「どう作るか」はそちらの領分です。設計図を引く者と、鉄を打つ者は、役割が違います。

## 前回どこまで掘ったかを憶えている

トランスクリプトは膨大です。数百メガバイトに達することもあります。毎回すべてを読み直すのは無駄で、同じ候補を何度も出すことにもなります。

鍛冶場は `~/.claude/idea-forge/state.json` に「前回どこまで掘ったか」のカーソルを持ちます。`history.jsonl` はどこまでの行を読んだか、トランスクリプトはどのセッションまで見たかを憶えていて、次の `dig` はその続きから掘ります。掘る量には上限があるので、履歴が積み上がっていても、一回の実行が暴走することはありません。

## できないこと・正直な制約

- **鍛冶場は作りません。** 候補を提案するだけです。実際にプラグインを組むのは `pluginize` や `plugin-dev` の仕事で、鍛冶場はそこに渡すための設計図を用意するところで止まります。
- **候補は候補です。** 履歴からパターンを読むのは判断を伴う作業で、たまたま繰り返しただけの偶然を「型」と誤読することもあります。根拠は添えますが、それが本当に作る価値のある自動化かを決めるのは、あなたです。
- **蓄積が薄いと、掘っても出ません。** 使い始めたばかりで履歴が浅ければ、地層の縞模様はまだできていません。候補が出ないのは、鍛冶場の不調ではなく、井戸がまだ浅いということです。時間が水を溜めます。
- **抽出には `claude` CLI を使います。** トランスクリプトからの手作業検出は軽量なサブエージェント（`claude -p`）に分割して行うので、`claude` がパスにあることが前提です。API キーは要りません。Claude Code の認証に相乗りします。
- **履歴は掘るだけ。外へは出しません。** 鍛冶場が読むのはすべてローカルの `~/.claude/` 配下で、外部サービスへは何も送りません。

## 構成

```
idea-forge/
├── .claude-plugin/
│   └── plugin.json
├── commands/
│   └── dig.md              # 四つの地層を掘り、重複を除外し、根拠の濃い順に候補を提案
├── skills/
│   └── forge-candidate/
│       ├── SKILL.md         # 候補の適合判定と、根拠の濃度で並べる規律
│       └── references/
│           └── fit-criteria.md   # プラグイン部品で実装可能か・既存重複でないか等の判定基準
└── README.md
```
