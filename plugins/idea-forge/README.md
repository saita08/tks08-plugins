# idea-forge

> あなたが去年からこっち、同じ形のプロンプトを何百回打ってきたか、あなたは知りません。
> どの手順を、毎回手で回しているか。何曜日の何時に、いつも同じ作業に戻ってくるか。
> その記録は、`~/.claude/` の炉の底に、火も入れられないまま冷たく積もっています。

自分の Claude 利用履歴は、次に鍛つべき道具の、いちばん正直な素材です。あなたが本当に何を繰り返しているかは、頭の中の思い込みではなく、その鉄鉱石の中に含まれています。ただ、誰も炉にくべない。`idea-forge` は、冷えた鉱石に火を入れ直す鍛冶場です。

## これは何？

`pluginize` という姉妹の職人は、**いま目の前で熱い作業**から「これ道具にできますよ」と拾います。作業台の脇に立って、赤いうちに鎚を入れる現場の鍛冶です。

`idea-forge` が炉にくべるのは、そこではありません。**冷えて積もった履歴の鉱脈**です。今日の一打ではなく、この数ヶ月あなたが掘り溜めてきた利用の露頭。そこには、一日ぶんの火では見えない不純物の縞が眠っています。毎週やっている手作業、いつの間にか癖になったコマンドの組み合わせ、特定の時間帯にいつも戻ってくる作業。一回では地金に紛れて見えないものが、百回分を炉にくべて溶かすと、鉱石の縞になって浮きます。

鍛冶場は、その縞を読み、プラグイン候補を**火で炙った証拠つきで**鍛え上げて差し出します。「なんとなく便利そう」という生の鉄ではなく、「あなたはこの形のプロンプトを41回打っていて、うち大半は同じ三手順で、既存のどの道具もこれを打てていない」という、鉱石の分析に裏打ちされた地金です。

## 何を炉にくべるのか

鍛冶場が火を入れる鉱脈は、四つあります。

- **コマンドの頻度とプロンプトの型** — `~/.claude/history.jsonl` に残るあなたの入力の全鉱床。どのスラッシュコマンドを何回、どんな形のプロンプトを繰り返し打っているか。同じ骨格のプロンプトが何十回も炉に現れれば、それはもう道具に鍛えられたがっている鉄です。
- **利用のリズム** — `~/.claude/stats-cache.json` に集計された、日ごと・時間帯ごとの活動。いつ、どれくらい、どのモデルで鎚を振っているか。リズムの刻みには、定期作業の熱が映ります。
- **繰り返しの手作業** — 直近トランスクリプトのサンプリングから、手で回している段取りや、毎回同じ順で叩いている手順を拾います。同じ形に何度も打ち直しているなら、それは型にすべき仕事です。
- **すでに持っている道具** — 導入済みのプラグインと組み込み機能。候補がこれらと被っていないかを鍛接面で突き合わせ、「もう棚にある道具」を提案から外します。炉に火を入れて、すでに壁に掛かっている鎚をもう一本鍛え直すのは、鍛冶場の恥です。

## 使い方

`/idea-forge:dig` を実行してください。鍛冶場は前回火を落としたところから鉱脈を掘り、四つの縞を炉にくべ、候補を絞り、**地金の詰まった順に**並べて差し出します。

各候補には、なぜそれが地金になったのかの証拠がつきます。どの鉱脈の、どんな縞から溶け出したのか。頻度なのか、リズムなのか、手作業の繰り返しなのか。よく詰まった鋼ほど上に来ます。薄く延ばした思いつきは、下に沈むか、そもそも炉から出てきません。

## 鍛冶場は地金を打つ。刃をつけるのは別の職人

鍛冶場の仕事は、候補を地金に鍛えて差し出すところまでです。**道具そのものは仕上げません。**

刃をつける段になったら、専門の職人に渡してください。目の前の熱い作業から拾う `pluginize`、あるいは対話で一挺ずつ組み上げる `plugin-dev`。鍛冶場は「何を打つべきか」を証拠つきで示す係で、「どう研ぎ上げるか」はそちらの領分です。地金を鍛える炉と、刃を研ぐ砥石は、火の役割が違います。

## 前回どこまで掘ったかを憶えている

トランスクリプトは、掘れば掘るほど深くなる坑道です。数百メガバイトに達することもあります。毎回坑道を入口から掘り直すのは無駄で、昨日溶かした鉱石をもう一度炉にくべることにもなります。

鍛冶場は `~/.claude/idea-forge/state.json` に「前回どこまで掘り進んだか」の刻み目を残します。`history.jsonl` はどの行まで読んだか、トランスクリプトはどのセッションまで炉にくべたかを憶えていて、次の `dig` はその刻み目の先から掘ります。一度に炉にくべる量には上限があるので、鉱脈がどれだけ深くても、一回の `dig` で炉が焼き切れることはありません。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/idea-forge:dig` | 四つの鉱脈を掘り、既存の道具と鍛接面で突き合わせて重複を外し、地金の詰まった順に候補を提案 |
| Skill | `forge-candidate` | 候補が地金として立つかの適合判定と、証拠の詰まり具合で並べる作法 |

Hook は使いません。鍛冶場はあなたが `dig` と呼んで火を入れたときにだけ炉を焚きます。呼ばれてもいないのに勝手に炉を焚き続ける鍛冶場は、鍛冶場ではなく火事です。

## よくある質問

**Q: `pluginize` があるのに、なぜもう一つ要るんですか？**
A: くべる素材が違います。`pluginize` は目の前で赤く熱している作業を拾い、鍛冶場は冷えて積もった数ヶ月の鉱脈を掘ります。今日の一打では見えない縞は、坑道の奥にしかありません。同じ炉に見えて、火を入れる場所が違う二挺です。

**Q: 出てきた候補は、そのまま作っていいものですか？**
A: 地金は地金であって、完成品ではありません。鉱石からパターンを読むのは判断の伴う仕事で、たまたま繰り返しただけの不純物を「型」と読み違えることもあります。証拠は炙って添えますが、それが本当に打つ価値のある道具かを決めるのは、あなたの目利きです。

**Q: 使い始めたばかりです。掘れますか？**
A: 掘っても、まだ何も溶け出しません。履歴が浅ければ、鉱脈の縞はまだできていない。候補が出ないのは鍛冶場の不調ではなく、坑道がまだ浅いということです。鉱石は時間が積みます。しばらく使い込んでから、また炉に来てください。

**Q: この鍛冶場、動くのに何が要りますか？**
A: 手作業の検出は軽量なサブエージェント（`claude -p`）に分けて行うので、`claude` がパスにあることが前提です。API キーは要りません。Claude Code の認証に相乗りします。炉にくべるのはすべてローカルの `~/.claude/` 配下の鉱石で、外の炉へは火の粉ひとつ飛ばしません。

---

いちばん正直な設計図は、あなたが自分で気づかないうちに、もう百回描いています。鍛冶場は、その線を炉から拾い上げるだけです。
