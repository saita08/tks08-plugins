# shikigami

> 「主が寝ている間、誰が門を守るのか」
> 「あなたのClaudeです」
> 「私のClaudeは、私が見ている時しか存在しないが」
> 「だから式神を放つのです」

ほとんどの人のClaudeは、画面を開いている間だけ生きています。閉じれば消え、朝には昨日の続きを覚えていません。それはアシスタントであって、インフラではありません。`shikigami` は、あなたが見ていない時間にも務めを果たす使い魔を、安全柵つきで一体ずつ放つプラグインです。陰陽師は夜回りを自分でしません。紙の式神を飛ばして、朝にその報告を読むのです。

## これは何？

式神とは、決まった刻限に自分で起きて、与えられた務めを果たし、必ず報告を残して眠る、自律したルーチンです。夜のうちに全リポジトリを巡回する式神、週末に一週間を収穫して振り返り草案を置いていく式神、毎朝あなたが見たい定点だけを並べる式神、特定のURLやリポジトリを見張り続ける式神。どれもあなたが不在の間に働き、朝、あなたは報告だけを読みます。

肝心なのは、式神が魔法ではないことです。式神はcronです。刻限に起きられるかどうかは、あなたの環境がどんなスケジューラを持っているかで決まります。だから `shikigami` は、放つ前に必ずスケジューラを探し、見つからなければ「この環境には届く仕掛けがありません」と正直に立ち止まります。守っていない持ち場を守っていると思い込ませるのが、一番罪深い嘘だからです。

## 五つの掟

式神は主の見ていない場所で働きます。だから放つ前に柵で囲います。使い魔は五つの掟に縛られていて、これは飾りではなく、暗がりで働く者を安全にする唯一の方法です。

**取り消せることしかしない。** 式神は読み、調べ、集め、考えます。何か「やるべきこと」を見つけたら、それを実行するのではなく、提案として報告書に書いて眠ります。実行はあなたが朝それを見て、承認してからです。暗がりで下した判断は取り消せません。提案なら、あなたが一言「やらない」と言えば消えます。

**務めと予算を持つ。** 一体の式神は一つの務めのために誂えられ、その外には手を出しません。隣に面白そうな仕事があっても、気づいて報告するのは務めのうち、手を出すのは務めの外です。務めには使えるトークンの上限がついています。誰も見ていない間に際限なく走る自律ルーチンは、それ自体が負債だからです。

**報告を書かない式神は存在しない。** 起きるたびに、必ず報告を残します。何も見つからなかった夜も「起きて、何もなかった」と書きます。見えない働きは、働いていないことと、間違って働いたことと、区別がつきません。報告ファイルの沈黙は「式神が起きなかった」だけを意味し、「起きたが黙った」を決して意味しない。ここが式神を信頼できる根拠です。

**いつでも解任できる。** 使い魔は主の意のままに、ためらいも抵抗もなく、その場で放免できます。安く解任できることが、安心して召喚できる理由でもあります。ここで召喚するものは、取り消せない約束を一つも含みません。

**同じ報告を繰り返す式神は、自ら解任を願い出る。** 毎晩同じ「異常なし」を書き続ける見張りは、あなたを報告から目を逸らさせます。そして目を逸らした瞬間、本当に何かを見つけた一度の報告も見過ごされます。見張り役に目を逸らさせる見張りは、見張りがいないより悪い。だから式神は自分の報告も見張り、壁紙になったと気づいたら、自分の解任を進言します。

## 使い方

一体、対話で誂えます。

```
/shikigami:summon
```

務めを選び、刻限を決め、五つの柵を一つずつ確認し、スケジューラに登録するところまでを、一問一答で進めます。柵の確認が済むまで登録には進みません。ここが関所です。自律ルーチンを野に放って安全かを決めるのは、道具ではなくあなただからです。

放った式神たちの点呼はこちら。

```
/shikigami:muster
```

いま務めに就いている使い魔の一覧と、それぞれの最新の報告を並べます。あなたが留守にしている間に起きたことが、ここに集まります。手を引いてくる報告があれば、それを先頭に出します。

用済みになったら、名指しで解任します。

```
/shikigami:dismiss <名>
```

スケジューラから firing を消し、登録簿から名を削ります。両方を同時に外さないと、登録簿から消えたのに走り続ける亡霊か、走っていないのに点呼に並ぶ幻のどちらかが残ります。

## 置き場所

式神たちは `~/.claude/shikigami/` に住みます。登録簿は `register.md` に、各自の報告は `reports/<名>.md` に、起きるたび日付つきで**追記**されます。どれも人間が読めるMarkdownです。見えない私的メモリには何も書きません。式神の思想は「見える場所に置く」ことそのものだからです。

## コンポーネント

| 種別 | 名前 | 説明 |
|------|------|------|
| Command | `/shikigami:summon` | 式神を一体、対話で誂えてスケジューラに登録する |
| Command | `/shikigami:muster` | 現役の式神を点呼し、各自の最新報告を並べる |
| Command | `/shikigami:dismiss` | 式神を解任し、スケジュールと登録簿の両方から外す |
| Skill | `shikigami-craft` | 式神の掟と、務めの雛形。Claudeが自律ルーチンを組む前に読む知識ベース |

## よくある質問

**Q: 本当に私が寝ている間に動くんですか？**
A: あなたの環境が刻限に起きる仕掛けを持っていれば、動きます。持っていなければ、`/shikigami:summon` は正直にそう言って、手渡し用のcron行を渡して立ち止まります。式神はcronの上に立つ紙人形であって、cronそのものを生み出す呪術ではありません。

**Q: スケジューラは見つかったのに、セッションを閉じたら式神が消えました。**
A: スケジューラには、セッションが生きている間だけ式神を預かるものと、機械そのものに刻む本物とがあります。前者は数日で自然に失効することもあります。`/shikigami:summon` はどちらかを見分けて登録簿に書き留めるので、点呼のときに「これは何日で消える式神か」まで分かります。守っていない持ち場を守っていると思わせないためです。

**Q: 式神が勝手にプルリクをマージしたり、どこかにメッセージを送ったりしませんか？**
A: しません。第一の掟がそれを禁じています。式神は見つけたことを提案として報告書に書くだけで、取り消せない行為は一切あなたの承認待ちになります。あなたの機械の外へ何かを送ることも、暗がりの firing では起きません。

**Q: 何体まで放てますか？**
A: 上限は設けていませんが、`/shikigami:summon` は一度に一体ずつしか誂えません。まとめて召喚した使い魔の群れは、あなたが後から理由を辿れない群れだからです。

**Q: 解任した式神の報告書は消えますか？**
A: 残します。それは使い魔が仕えていた間の働きの記録であり、解任は歴史を消す理由にはなりません。消すかどうかは、あなたが決めてください。

放った式神は、あなたが忘れても、刻限を忘れません。それが使い魔と、締め切りに追われる人間の違いです。
